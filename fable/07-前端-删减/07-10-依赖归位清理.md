# 07-10 依赖归位清理

## 1. 原始需求

对 `apps/web/package.json` 逐个依赖 grep 验证 import 后，发现三类问题：

1. **归位错误**：`@types/nprogress`（第 36 行）在 `dependencies` 里。类型包只在编译期使用，应放 `devDependencies`（`nprogress` 本体在 dependencies 是对的，`RouteProgressBar.tsx:3` 运行时引用）。
2. **零引用依赖**（全库 `rg "from '<包名>"` 均为 0 命中，且非配置文件隐式加载）：
   - `class-variance-authority`（dependencies 第 39 行）——shadcn 模板常见依赖，但本项目 UI 封装（如 `shared/components/ui/button.tsx`）是手写 variant 映射对象，没有用 `cva()`；全库无 import。
   - `msw`（devDependencies 第 67 行）——测试 mock 全部使用 `vi.mock`（97 个测试文件均未 import msw），无 handlers/server 设置文件。
3. **说明现状、不迁移**：`electron`（devDependencies 第 60 行）在 apps/web 下服务的是 `desktop:timer` 脚本（`electron ../desktop-timer/main.cjs`），运行的却是隔壁 `apps/desktop-timer/` 的代码（该目录只有 main.cjs/preload.cjs，无自己的 package.json）。这属于结构怪味但能用，本文档**只记录，不强制迁移**。

其余依赖均验证有真实引用，不在本文档处理范围：所有 @radix-ui/*（各 1~2 文件，ui 封装引用）、`@tanstack/react-query`（3 文件，去留见 07-01）、`@xyflow/react`（5）、`canvas-confetti`（1）、`clsx`（1）、`cmdk`（1）、`lucide-react`（76）、`motion`（2）、`react-router-dom`（49）、`recharts`（2）、`sonner`（4）、`tailwind-merge`（1）、`tailwindcss` + `@tailwindcss/vite`（vite.config.ts:3 插件 + index.css:1 `@import "tailwindcss"`）、`openapi-typescript`（脚本引用，处理归 07-05）。
