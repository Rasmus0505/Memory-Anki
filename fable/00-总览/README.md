# fable — Memory Anki 全方位改进设计文档库

本文件夹是对 Memory-Anki 项目（记忆宫殿 + 间隔重复 + AI 辅助学习的个人自用产品）从**代码架构、功能板块、前端设计**三个维度提出的**删减 / 优化 / 新增**全方位改进方案。

每篇文档 = 一个独立可执行的改进任务，包含：

1. **原始需求**：为什么做，现状证据（真实文件路径）。
2. **详细执行清单**：面向弱 AI 编程模型的傻瓜式步骤，逐文件逐步骤，含代码示意与"不要做什么"。
3. **测试验收标准**：可执行命令 + 人工操作验收 + 回归检查。
4. **进度记录**：执行者交接表，随做随记。

## 使用方式（给执行代理）

1. 先读 `00-总览/文档模板.md` 了解格式与硬性约束。
2. 打开 `00-总览/总索引与进度看板.md`，认领"未开始"且依赖已满足的文档。
3. 把该文档"状态"改为"进行中"并填负责代理，同时更新总索引对应行。
4. 严格按执行清单逐步执行，每步做完核对自查点。
5. 跑完测试验收标准全部命令后，把状态改为"已完成"，在进度记录表追加一行。

## 硬性约束

- 严格遵守仓库根目录 `1.md`（Karpathy 行为准则）：先想后写、最小改动、外科手术式修改、目标驱动验证。
- 严格遵守 `AGENT.md`：跨设备可用（两台 Windows 电脑）、不写死本机路径、PWA 只用现有后端。
- 严格遵守 `AI_PROJECT_CONTEXT.md` 第 6 节架构边界。
