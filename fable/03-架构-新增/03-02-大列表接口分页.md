# 03-02 大列表接口分页

## 1. 原始需求

以下接口当前一次性全量返回，随数据积累（宫殿数百个、学习会话数千条、题库上千题）会拖慢首屏与手机端流量：

- `GET /palaces`：`apps/api/src/memory_anki/modules/palaces/presentation/router.py` 第 112~114 行，`list_palaces` 无任何数量限制，返回裸数组；
- `GET /study-sessions`：`apps/api/src/memory_anki/modules/sessions/presentation/router.py` 第 95~99 行，`list_study_sessions`（`study_session_service.py` 第 329~338 行 `query.all()`）无 limit；
- `GET /subjects`：`apps/api/src/memory_anki/modules/knowledge/presentation/router.py` 第 66~68 行，返回裸数组；
- `GET /palaces/{id}/quiz-questions` 与 `GET /palaces/{id}/aggregated-quiz-questions`：`apps/api/src/memory_anki/modules/palace_quiz/presentation/router.py` 第 79~83、98~102 行，全量返回 `{items}`。

目标：统一增加**可选** `limit`/`offset` 查询参数；**不传参数时响应结构与内容与现状逐字节兼容**（渐进方案，前端可分批接入）；传 `limit` 时返回带 `total` 的响应包。

## 2. 详细执行清单

约定（四个接口统一）：

- `limit: int | None = None`、`offset: int = 0`，用 FastAPI Query 校验 `limit >= 1`、`offset >= 0`，`limit` 上限 500；
- `limit` 为 `None` → 保持现有返回结构（裸数组或 `{items}`）；
- `limit` 有值 → 返回 `{"items": [...], "total": <过滤后总数>, "limit": limit, "offset": offset}`。

### 步骤 1：`GET /study-sessions`（最简单，先做）

1a. 打开 `apps/api/src/memory_anki/modules/sessions/application/study_session_service.py`，找到第 329 行 `def list_study_sessions(...)`，修改为：

```python
# 修改前
def list_study_sessions(
    session: Session,
    *,
    include_deleted: bool = False,
    include_below_threshold: bool = False,
) -> list[dict[str, Any]]:
    query = session.query(StudySession).order_by(StudySession.started_at.desc())
    query = query.filter(StudySession.deleted_at.is_(None))
    rows = query.all()
    return [study_session_json(row) for row in rows]

# 修改后
def list_study_sessions(
    session: Session,
    *,
    include_deleted: bool = False,
    include_below_threshold: bool = False,
    limit: int | None = None,
    offset: int = 0,
) -> list[dict[str, Any]]:
    query = session.query(StudySession).order_by(StudySession.started_at.desc())
    query = query.filter(StudySession.deleted_at.is_(None))
    if limit is not None:
        query = query.offset(max(0, offset)).limit(limit)
    rows = query.all()
    return [study_session_json(row) for row in rows]


def count_study_sessions(session: Session) -> int:
    return (
        session.query(StudySession)
        .filter(StudySession.deleted_at.is_(None))
        .count()
    )
```

1b. 打开 `apps/api/src/memory_anki/modules/sessions/presentation/router.py`，修改第 95~99 行：

```python
# 修改前
@router.get("/study-sessions")
def api_list_study_sessions(
    session: Session = Depends(session_dep),
):
    return {"items": list_study_sessions(session)}

# 修改后
@router.get("/study-sessions")
def api_list_study_sessions(
    limit: int | None = Query(default=None, ge=1, le=500),
    offset: int = Query(default=0, ge=0),
    session: Session = Depends(session_dep),
):
    if limit is None:
        return {"items": list_study_sessions(session)}
    return {
        "items": list_study_sessions(session, limit=limit, offset=offset),
        "total": count_study_sessions(session),
        "limit": limit,
        "offset": offset,
    }
```

同时在文件顶部 `from fastapi import APIRouter, Depends, HTTPException` 中加入 `Query`，并把 `count_study_sessions` 加进第 28~42 行的 service import 列表。不要动同文件其他路由（尤其 `/study-sessions/active`、`/stats` 的顺序在 `/{study_session_id}` 之前，保持不变）。

**自查点**：`cd apps/api && python -m pytest tests/test_study_session_routes.py -q` 通过；`curl "http://127.0.0.1:8012/api/v1/study-sessions"` 返回结构不含 `total`，`curl "...?limit=5"` 含 `total`。

### 步骤 3：`GET /subjects`

打开 `apps/api/src/memory_anki/modules/knowledge/presentation/router.py`，第 66~68 行：

```python
# 修改前
@router.get("/subjects")
def list_subjects(s: Session = Depends(session_dep)):
    return [subject_json(sub) for sub in s.query(Subject).order_by(Subject.sort_order).all()]

# 修改后
@router.get("/subjects")
def list_subjects(
    limit: int | None = Query(default=None, ge=1, le=500),
    offset: int = Query(default=0, ge=0),
    s: Session = Depends(session_dep),
):
    query = s.query(Subject).order_by(Subject.sort_order)
    if limit is None:
        return [subject_json(sub) for sub in query.all()]
    items = [subject_json(sub) for sub in query.offset(offset).limit(limit).all()]
    return {"items": items, "total": s.query(Subject).count(), "limit": limit, "offset": offset}
```

### 步骤 4：宫殿题库两个列表接口

打开 `apps/api/src/memory_anki/modules/palace_quiz/presentation/router.py`。第 79~83 行与第 98~102 行的两个路由采用相同模式：在路由签名增加 `limit`/`offset` Query 参数；`limit is None` 时保持 `{"items": list_questions(s, palace_id)}` 原样；有值时对 service 返回的列表做 Python 层切片并回填 total：

```python
# 修改后（以 quiz-questions 为例，aggregated-quiz-questions 同样处理）
@router.get("/palaces/{palace_id}/quiz-questions")
def api_list_palace_quiz_questions(
    palace_id: int,
    limit: int | None = Query(default=None, ge=1, le=500),
    offset: int = Query(default=0, ge=0),
    s: Session = Depends(session_dep),
):
    try:
        items = list_questions(s, palace_id)
        if limit is None:
            return {"items": items}
        return {
            "items": items[offset : offset + limit],
            "total": len(items),
            "limit": limit,
            "offset": offset,
        }
    except Exception as exc:  # pragma: no cover - centralized HTTP mapping
        ...  # 保持既有异常映射不动
```

说明：题目列表按宫殿聚合后通常在千条以内，且 `list_questions`/`list_aggregated_questions` 内部有排序与拼装逻辑，第一步用内存切片最稳妥；后续若有性能需求再下推到 SQL，不属于本文档。不要改动 dedupe、batch 等其他路由。

**自查点**：`cd apps/api && python -m pytest tests/test_palace_quiz_routes.py -q` 无新增失败。
