# 03-04 附件静态目录访问鉴权

## 1. 原始需求

`apps/api/src/memory_anki/app/main.py` 第 153 行：

```python
app.mount("/api/attachments", StaticFiles(directory=str(ATTACHMENTS_DIR)), name="attachments")
```

把整个附件目录（`%LOCALAPPDATA%\MemoryAnki\data\attachments`，含 `subjects/` 子目录的学科文档，见 `core/config.py` 第 113~114 行）以静态文件形式整体暴露。经 Tailscale Serve 暴露到 tailnet 后，任何 tailnet 设备只要拿到/猜到文件名即可无凭据直接下载，无任何校验。

核实补充（写文档时已验证）：

- 前端实际使用的是**带 DB 校验的路由** `GET /api/v1/attachments/{att_id}`（`modules/palaces/presentation/router.py` 第 609~615 行：先查 `attachments` 表再 `FileResponse`），`apps/web/src/entities/palace/api/catalogApi.ts` 第 49~51 行 `buildAttachmentUrl` 也指向 `/api/v1/attachments/{id}`；
- 全仓 grep `"/api/attachments"`（不含 `/api/v1/attachments`）**仅命中 main.py 这一处挂载**，未发现任何前端消费者——该挂载疑似历史遗留。

目标：让 `/api/attachments/*` 不再匿名可读。首选方案 A（零代码，依赖 03-01 的令牌中间件按 `/api` 前缀覆盖）；提供方案 B（核查确认无消费者后直接移除挂载）作为可选收尾；提供方案 C（03-01 长期不实施时的独立兜底）。

## 2. 详细执行清单

### 方案 A（首选）：由 03-01 令牌中间件覆盖

原理：Starlette 中 `add_middleware` 注册的中间件包裹整个应用，在路由分发（含 `app.mount` 的子应用）**之前**执行；03-01 的 `ApiTokenAuthMiddleware` 按 `path.startswith("/api")` 拦截，因此 `/api/attachments/*` 自动被覆盖，无需为本文档写任何生产代码。本方案只做验证与测试固化。

#### 步骤 A1：确认中间件覆盖挂载路径

前置条件：03-01 已实施。在 `.env` 配置 `MEMORY_ANKI_API_TOKEN=abc123` 并重启服务，然后执行：

```powershell
# 1) 模拟远程请求（带转发头、无令牌）→ 期望 401
curl.exe -s -o NUL -w "%{http_code}`n" -H "X-Forwarded-For: 100.64.0.9" http://127.0.0.1:8012/api/attachments/whatever.png

# 2) 模拟远程请求（带令牌）→ 期望 200（文件存在）或 404（文件不存在）
curl.exe -s -o NUL -w "%{http_code}`n" -H "X-Forwarded-For: 100.64.0.9" -H "X-Memory-Anki-Token: abc123" http://127.0.0.1:8012/api/attachments/whatever.png

# 3) 本机直连（无转发头）→ 期望 200/404（回环豁免）
curl.exe -s -o NUL -w "%{http_code}`n" http://127.0.0.1:8012/api/attachments/whatever.png
```

**自查点**：三条命令依次返回 401、404（或 200）、404（或 200）。若第一条不是 401，说明 03-01 中间件的路径判断或挂载顺序有误，回到 03-01 排查，不要在本文档打补丁。

### 方案 C（兜底，仅当决定长期不做 03-01 时）：独立的挂载守卫

若 03-01 被搁置，可用一个只护住该挂载的最小包装（与 03-01 互斥，两者都做会重复校验）。新建 `apps/api/src/memory_anki/core/guarded_static.py`：

```python
from __future__ import annotations

from starlette.exceptions import HTTPException as StarletteHTTPException
from starlette.staticfiles import StaticFiles


class LoopbackOnlyStaticFiles(StaticFiles):
    """仅允许本机直连读取的静态目录（Tailscale 转发请求一律 403）。"""

    async def __call__(self, scope, receive, send):
        headers = {k.decode().lower(): v for k, v in scope.get("headers", [])}
        client = scope.get("client")
        is_loopback = (
            client is not None
            and client[0] in {"127.0.0.1", "::1"}
            and b"x-forwarded-for" not in [k for k, _ in scope.get("headers", [])]
        )
        _ = headers  # headers 变量供调试；判断以原始列表为准
        if not is_loopback:
            raise StarletteHTTPException(status_code=403)
        await super().__call__(scope, receive, send)
```

main.py 第 153 行改为：

```python
from memory_anki.core.guarded_static import LoopbackOnlyStaticFiles  # import 区

app.mount(
    "/api/attachments",
    LoopbackOnlyStaticFiles(directory=str(ATTACHMENTS_DIR)),
    name="attachments",
)
```

代价：手机端将完全无法读该目录（连带令牌也不行）——鉴于 B1 核查显示无消费者，这个代价可接受。方案 C 与方案 A 二选一，**默认走 A**。

**自查点**：本机 curl 返回 200/404，带 `X-Forwarded-For` 的 curl 返回 403。

### 明确不要做什么

- 不要新写"带校验的逐文件响应路由"替代静态挂载——`GET /api/v1/attachments/{att_id}`（palaces router 第 609 行）已是带 DB 校验的文件响应路由且前端在用，重复造轮子属过度设计；
- 不要给 `/api/v1/attachments/{att_id}` 增加额外权限模型（单用户产品，03-01 令牌中间件已覆盖全部 `/api` 前缀）；
- 方案 B 在 B1 核查不通过时坚决不做；方案 C 只在 03-01 明确搁置时做；
- 不要改动 `SUBJECT_DOCUMENTS_DIR` 等目录结构或附件上传逻辑。

1. 配置令牌后，手机（经 Tailscale）直接打开 `https://<host>/api/attachments/<真实文件名>` → 401；带令牌请求头 → 200。

3. （若执行方案 B）访问 `/api/attachments/任意路径` → 404，且宫殿附件上传/预览/删除全链路正常。
4. （若执行方案 C）本机可读、远程 403。
