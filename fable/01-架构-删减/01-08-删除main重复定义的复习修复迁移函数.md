# 01-08 删除 main.py 重复定义的复习修复迁移函数

## 1. 原始需求

`run_review_schedule_repair_migration` 在两个文件中逐行重复定义（已逐行比对，逻辑完全一致，
仅 `session` 参数的类型标注有无之差）：

```88:98:apps/api/src/memory_anki/app/main.py
def run_review_schedule_repair_migration(session):
    if is_app_migration_completed(REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY):
        return None
    result = repair_review_stage_progress(session)
    mark_app_migration_completed(
        REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY,
        {
            "result": result,
        },
    )
    return result
```

```63:73:apps/api/src/memory_anki/app/startup_runtime.py
def run_review_schedule_repair_migration(session: Session):
    if is_app_migration_completed(REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY):
        return None
    ...
```

**真正在启动链路上被调用的是 startup_runtime 版本**（`startup_runtime.py` 第 96 行
`run_prepare_runtime()` 内调用）；main.py 版本在生产代码中无人调用，只有
`apps/api/tests/test_review_routes.py` 的两个测试（第 2974-3018 行）通过 `main_module.` 前缀
调用它并猴补丁 `main_module.is_app_migration_completed` 等三个符号。main.py 为这份重复函数拖着
三组 import（`REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY`、`core.migration` 两个函数、
`repair_review_stage_progress`），删除函数后这些 import 也随之无用。

目标：删除 main.py 的重复定义与陪葬 import，测试改为针对 startup_runtime 模块。

```python
    def test_run_review_schedule_repair_migration_marks_completion_once(self):
        original_is_completed = startup_runtime_module.is_app_migration_completed
        original_mark_completed = startup_runtime_module.mark_app_migration_completed
        original_repair = startup_runtime_module.repair_review_stage_progress
        calls: list[tuple[str, dict]] = []

        try:
            startup_runtime_module.is_app_migration_completed = lambda key: False
            startup_runtime_module.repair_review_stage_progress = lambda session: {"palace_count": 1, "segment_count": 2}
            startup_runtime_module.mark_app_migration_completed = lambda key, payload: calls.append((key, payload))

            with self.SessionLocal() as session:
                result = startup_runtime_module.run_review_schedule_repair_migration(session)

            self.assertEqual(result, {"palace_count": 1, "segment_count": 2})
            self.assertEqual(len(calls), 1)
            self.assertEqual(calls[0][0], startup_runtime_module.REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY)

            ...
```

`from ... import is_app_migration_completed` 等方式把三个函数绑定为**本模块属性**，
测试替换 `startup_runtime_module.<名字>` 恰好命中函数体内的查找路径，机制与原来补丁 main_module 相同。
