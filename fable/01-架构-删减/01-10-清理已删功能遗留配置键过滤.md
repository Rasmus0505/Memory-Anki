# 01-10 清理已删功能遗留配置键过滤

## 1. 原始需求

`apps/api/src/memory_anki/modules/settings/presentation/router.py` 的 `read_settings()`（第 69-88 行）在读取 `config` 表时硬编码排除 12 个遗留键：

```python
        if row.key in {
            "default_algorithm",
            "algorithm_change_scope",
            "custom_intervals",
            "time_recording_threshold_seconds",
            "flow_voice_api_key",
            ...  # flow_voice_* 共 8 个
        }:
            continue
```

这些键属于已删除的功能（多算法选择、时间记录阈值、语音朗读 flow_voice）。经核实（2026-07-08）：

- 全后端 `rg "flow_voice|default_algorithm"` 仅命中此排除名单与 `schedule_rebuild_service.py`/`segment_review_support.py` 的 `default_algorithm` 死参数（后者由 01-05 处理）——即**没有任何代码再写入这些键**，`core/config.py` 的 `DEFAULTS`（第 140-196 行）里也没有它们；
- 历史迁移 `apps/api/alembic/versions/0008_prune_deleted_features.py` 第 94-106 行已从 `config` 表删过其中 3 个键（`time_recording_threshold_seconds`、`default_algorithm`、`algorithm_change_scope`），但**漏掉了 `custom_intervals` 与 8 个 `flow_voice_*` 键**，所以 read_settings 仍要靠排除名单兜底；
- 当前最新迁移是 `0012_freestyle_history.py`。

正确做法：数据在 DB 层清干净（新增 0013 清理迁移），代码层的排除名单随之整体删除。

## 2. 详细执行清单

> 共修改 1 个文件 + 新增 1 个迁移文件。顺序不能反：必须先落地迁移，再删排除名单。不要动 `write_settings`、client_preferences 相关代码。

### 步骤 1：新增清理迁移 0013

新建文件 `apps/api/alembic/versions/0013_prune_legacy_config_keys.py`，完整内容如下（模式照抄 0008 第 93-106 行的 config 清理写法）：

```python
"""prune legacy config keys left behind by deleted features

Revision ID: 0013_prune_legacy_config_keys
Revises: 0012_freestyle_history

memory-anki: allow-destructive-migration
"""

from __future__ import annotations

import sqlalchemy as sa
from alembic import op

revision = "0013_prune_legacy_config_keys"
down_revision = "0012_freestyle_history"
branch_labels = None
depends_on = None


def _table_exists(table_name: str) -> bool:
    bind = op.get_bind()
    row = bind.exec_driver_sql(
        "SELECT name FROM sqlite_master WHERE type='table' AND name = ?",
        (table_name,),
    ).fetchone()
    return row is not None


def upgrade() -> None:
    if not _table_exists("config"):
        return
    op.get_bind().execute(
        sa.text(
            """
            DELETE FROM config
            WHERE key IN (
                'default_algorithm',
                'algorithm_change_scope',
                'custom_intervals',
                'time_recording_threshold_seconds',
                'flow_voice_api_key',
                'flow_voice_base_url',
                'flow_voice_model',
                'flow_voice_voice',
                'flow_voice_format',
                'flow_voice_sample_rate',
                'flow_voice_instruction',
                'flow_voice_thinking_enabled'
            )
            """
        )
    )


def downgrade() -> None:
    return
```

说明：键清单与 read_settings 排除名单（router.py 第 72-85 行）逐一对应，共 12 个；其中 3 个（`default_algorithm`、`algorithm_change_scope`、`time_recording_threshold_seconds`）0008 已删过，再删一次是幂等的，可兜住 0008 之后同步回来的旧库副本。执行迁移前先备份数据库（项目自带备份机制，或手工复制 `memory_palace.db`）。

自查点：`cd apps/api && python -m alembic upgrade head` 成功；随后 `python -c` 或 sqlite 客户端执行 `SELECT COUNT(*) FROM config WHERE key LIKE 'flow_voice%' OR key IN ('custom_intervals','default_algorithm','algorithm_change_scope','time_recording_threshold_seconds')` 返回 0。

### 步骤 2：删除 read_settings 的排除名单

打开 `apps/api/src/memory_anki/modules/settings/presentation/router.py`，`read_settings`（第 69-88 行）修改前：

```python
def read_settings(session: Session) -> dict:
    result = dict(DEFAULTS)
    for row in session.query(Config).all():
        if row.key in {
            "default_algorithm",
            "algorithm_change_scope",
            "custom_intervals",
            "time_recording_threshold_seconds",
            "flow_voice_api_key",
            "flow_voice_base_url",
            "flow_voice_model",
            "flow_voice_voice",
            "flow_voice_format",
            "flow_voice_sample_rate",
            "flow_voice_instruction",
            "flow_voice_thinking_enabled",
        }:
            continue
        result[row.key] = row.value
    return result
```

修改后：

```python
def read_settings(session: Session) -> dict:
    result = dict(DEFAULTS)
    for row in session.query(Config).all():
        result[row.key] = row.value
    return result
```

行为等价性依据：迁移已保证 DB 中不存在这 12 个键，且 `write_settings`（第 94-102 行）只写 `key in DEFAULTS` 的键、`DEFAULTS` 不含它们，故不会再产生新行。

不要做的事：

- 不要动 `SCHEDULE_IMPACTING_KEYS`（第 40-44 行）与 `CLIENT_PREFERENCE_GROUPS`（第 46-56 行）；
- 不要"顺手"过滤 `client_preferences.*` 行——它们当前就会出现在 `read_settings` 结果里，这是既有行为，改它超出本文档范围；
- 不要修改 `write_settings` 的白名单逻辑。

自查点：`rg -n "flow_voice|default_algorithm|algorithm_change_scope|custom_intervals|time_recording_threshold_seconds" apps/api/src/memory_anki/modules/settings` 无输出。

### 步骤 3：双设备注意事项（跨设备同步库）

本项目数据库经百度网盘在两台 Windows 设备间同步。alembic 迁移在任一台设备首次启动新代码时执行即可；但若另一台设备用旧代码写回了带遗留键的库副本，重跑 `alembic upgrade head` 不会重复执行 0013。因此验收前在**两台设备都拉取新代码并各自启动一次**，或手工确认同步后的库中遗留键计数为 0。

自查点：两台设备（或至少当前设备 + 对同步副本的抽查）满足步骤 1 的 SQL 断言。
