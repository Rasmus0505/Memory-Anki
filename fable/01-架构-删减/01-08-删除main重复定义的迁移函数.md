# 01-08 删除 main.py 重复定义的迁移函数

## 1. 原始需求

`run_review_schedule_repair_migration` 在两个文件中各定义了一份，经逐行对比核实（2026-07-08）两者逻辑完全一致（唯一差异是 startup_runtime 版带 `session: Session` 类型注解）：

- `apps/api/src/memory_anki/app/main.py` 第 88-98 行；
- `apps/api/src/memory_anki/app/startup_runtime.py` 第 63-73 行。

**生产链路只调用 startup_runtime 版**（`startup_runtime.py` 第 96 行 `run_prepare_runtime()` 内调用）；main.py 版在生产代码中零调用，唯一引用方是测试 `apps/api/tests/test_review_routes.py` 第 2974-3018 行的两个用例，通过 `main_module.run_review_schedule_repair_migration(session)` 调用并 monkeypatch `main_module` 上的三个协作函数。双定义的风险：将来改迁移逻辑时只改一份，测试测的还是没人跑的那份，形成假绿。

方案：删除 main.py 的副本及其陪葬 import，把两个测试用例改为指向 `startup_runtime` 模块（该模块第 11-15 行、第 28-30 行以相同名字 import 了 `is_app_migration_completed`/`mark_app_migration_completed`/`repair_review_stage_progress`，monkeypatch 模式可原样平移）。

## 2. 详细执行清单

> 共修改 2 个文件：`apps/api/src/memory_anki/app/main.py`、`apps/api/tests/test_review_routes.py`。不要动 `startup_runtime.py`（保留的那份定义一个字符都不要改）。

### 步骤 1：删除 main.py 的函数副本

打开 `apps/api/src/memory_anki/app/main.py`，整体删除第 88-98 行：

```python
def run_review_schedule_repair_migration(session):
    if is_app_migration_completed(REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY):
        return None
    result = repair_review_stage_progress(session)
    mark_app_migration_completed(
        REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY,
        {
            "result": result,
        },
    )
    return result
```

自查点：该文件中 `rg -n "run_review_schedule_repair_migration" src/memory_anki/app/main.py` 无输出。

### 步骤 2：删除因此不再使用的 import（三处）

仍在 `main.py`：

2a. 第 11-16 行，修改前：

```python
from memory_anki.app.startup_runtime import (
    REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY,
    STARTUP_MODE_SERVE,
    initialize_service_runtime,
    resolve_startup_mode,
)
```

修改后（`REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY` 在 main.py 只被被删函数使用，已核实）：

```python
from memory_anki.app.startup_runtime import (
    STARTUP_MODE_SERVE,
    initialize_service_runtime,
    resolve_startup_mode,
)
```

2b. 第 19-22 行，整条删除（`is_app_migration_completed`/`mark_app_migration_completed` 在 main.py 无其他使用，已核实）：

```python
from memory_anki.core.migration import (
    is_app_migration_completed,
    mark_app_migration_completed,
)
```

2c. 第 42-44 行，整条删除（`repair_review_stage_progress` 在 main.py 无其他使用，已核实）：

```python
from memory_anki.modules.reviews.application.review_execution_service import (
    repair_review_stage_progress,
)
```

注意：第 28 行 `from memory_anki.infrastructure.db.models import get_session as _get_session` 与第 49 行 `get_session = _get_session` **保留不动**（测试在多处 monkeypatch `main_module.get_session`，与本文档无关）。

自查点：`cd apps/api && python -m ruff check src` 通过（若有漏删的未使用 import 会报 F401）。

### 步骤 3：把两个测试用例改为指向 startup_runtime

打开 `apps/api/tests/test_review_routes.py`。

3a. 文件头部（第 15 行附近）现有 `import memory_anki.app.main as main_module`，**保留它**（其他用例还在用 `main_module.app`、`main_module.get_session`），在其下新增一行：

```python
import memory_anki.app.startup_runtime as startup_runtime_module
```

3b. 用例 `test_run_review_schedule_repair_migration_marks_completion_once`（第 2974-2995 行）：把该用例体内全部 `main_module.` 前缀替换为 `startup_runtime_module.`（共 9 处：3 处备份原函数、3 处 monkeypatch 赋值、1 处调用、1 处 `REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY` 断言、finally 里 3 处恢复——按实际出现逐个替换）。示意（修改后开头几行）：

```python
    def test_run_review_schedule_repair_migration_marks_completion_once(self):
        original_is_completed = startup_runtime_module.is_app_migration_completed
        original_mark_completed = startup_runtime_module.mark_app_migration_completed
        original_repair = startup_runtime_module.repair_review_stage_progress
        ...
            result = startup_runtime_module.run_review_schedule_repair_migration(session)
```

3c. 用例 `test_run_review_schedule_repair_migration_skips_when_already_completed`（第 2997-3018 行）：同样把用例体内 `main_module.` 全部替换为 `startup_runtime_module.`。

替换可行性依据（已核实）：`startup_runtime.py` 顶层以相同名字持有 `is_app_migration_completed`、`mark_app_migration_completed`（第 11-15 行）、`repair_review_stage_progress`（第 28-30 行）、常量 `REVIEW_SCHEDULE_REPAIR_MIGRATION_KEY`（第 46 行），monkeypatch 语义与原来完全一致。

不要做的事：不要把这两个用例之外的 `main_module.` 引用（如 `main_module.app`、`main_module.get_session`）改掉；不要重命名用例。

自查点：`python -m pytest tests/test_review_routes.py -k repair_migration` 两个用例通过。
