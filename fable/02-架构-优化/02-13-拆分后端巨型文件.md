# 02-13 拆分后端巨型文件

## 1. 原始需求

四个后端文件超过 600 行，超出单文件可维护上限（`ralph/prd.json` US-016~US-023 为前端确立的标准是"单文件 <350 行、按职责分文件、测试通过、typecheck 通过"，后端沿用同一精神）。实测行数：

| 文件 | 行数 |
|---|---|
| `apps/api/src/memory_anki/modules/palaces/application/mindmap_import/normalization.py` | 725 |
| `apps/api/src/memory_anki/modules/sessions/application/study_session_service.py` | 694 |
| `apps/api/src/memory_anki/modules/settings/application/ai_model_registry_admin.py` | 634 |
| `apps/api/src/memory_anki/modules/palaces/presentation/router.py` | 628 |

期望效果：每个文件拆分后所有产出文件 <350 行；**只移动不改写**；对外符号（被其他模块/测试 import 的名字）通过原文件 re-export 保持兼容，或同步更新引用。

## 2. 详细执行清单

> 禁止事项：严禁在移动的同时"顺手"改函数逻辑/重命名/调整默认参数；每个文件的拆分是独立任务，一次只做一个文件并单独跑全量测试；拆分 `normalization.py` 时不得触碰 `pyproject.toml` mypy 豁免（该模块在豁免名单，新文件若报 mypy 错，把新模块名**加入**同一豁免条目并在进度表记录，摘除交给 02-15）。

通用手法（四个文件都用）：
1. `rg -n "from <完整模块路径> import" apps/api/src apps/api/tests` 生成外部引用表；
2. 新建目标文件并移动函数（连同其专属 import 与模块级常量）；
3. 原文件（或包 `__init__.py`）保留 `from .<新文件> import <被外部引用的名字>  # re-export` 兼容行；
4. 跑 `python -m pytest tests -q`、`ruff check`、`lint-imports`。
