# 02-04 全局异常处理器与统一错误响应

## 1. 原始需求

后端目前**三种错误风格并存**，前端无法用统一逻辑判断失败：

1. 规范式：`raise HTTPException(status_code=..., detail=...)` —— sessions、freestyle、settings 等 router 采用。
2. 手写 JSONResponse：`apps/api/src/memory_anki/modules/knowledge/presentation/router.py` 153 行 `JSONResponse(status_code=409, content={"detail": str(exc)})`、166 行 `JSONResponse(status_code=500, content={"error": tb})`（把完整 traceback 返回给客户端）。
3. **200 + `{"error": ...}`**：knowledge router 84、97、110、**122**、**134**、**174** 行等对"not found"返回 HTTP 200 与 `{"error": "not found"}`；palaces router（152、167 行等）同样大量存在。HTTP 语义错误，前端必须逐接口判断 `resp.error`。

另外 `apps/api/src/memory_anki/app/main.py` 135-138 行，关机备份异常被静默吞掉：

```python
try:
    create_shutdown_backup()
except Exception:
    pass
```

期望效果：（a）应用级 `exception_handler` 把所有错误统一为 `{"code": str, "message": str, "detail": Any}` 结构；（b）未捕获异常返回 500 且**不泄露 traceback**（写日志）；（c）各 router 按迁移清单渐进改为抛异常而非返回 200+error；（d）关机备份失败至少记一条日志。

## 2. 详细执行清单

> 禁止事项：不要一次性改完所有 router（渐进迁移，每个 router 独立可回滚）；迁移某个 router 前**必须**先用 rg 检查前端对应 API 的错误处理方式（见步骤 4 的强制检查）；不要改动 SSE/流式端点（palace_quiz router 的 `_quiz_sse`）的错误通道；不要删除 knowledge router 里的 DEBUG 日志文件写入（157-164 行），只把"返回 tb 给客户端"改掉。

### 步骤 1：新建错误类型与处理器模块

新建 `apps/api/src/memory_anki/app/error_handlers.py`：

```python
"""全局异常处理：把所有错误统一为 {code, message, detail} JSON 结构。"""

import logging

from fastapi import FastAPI, Request
from fastapi.exceptions import RequestValidationError
from fastapi.responses import JSONResponse
from starlette.exceptions import HTTPException as StarletteHTTPException

logger = logging.getLogger(__name__)


def _error_body(code: str, message: str, detail=None) -> dict:
    return {"code": code, "message": message, "detail": detail}


def install_error_handlers(app: FastAPI) -> None:
    @app.exception_handler(StarletteHTTPException)
    async def handle_http_exception(request: Request, exc: StarletteHTTPException):
        detail = exc.detail
        if isinstance(detail, dict):
            # 兼容 settings router 现有的 {"message":..., "code":...} detail 结构
            code = str(detail.get("code") or f"http_{exc.status_code}")
            message = str(detail.get("message") or "")
            extra = {k: v for k, v in detail.items() if k not in {"code", "message"}}
            return JSONResponse(status_code=exc.status_code,
                                content=_error_body(code, message, extra or None))
        return JSONResponse(status_code=exc.status_code,
                            content=_error_body(f"http_{exc.status_code}", str(detail), None))

    @app.exception_handler(RequestValidationError)
    async def handle_validation_error(request: Request, exc: RequestValidationError):
        return JSONResponse(status_code=422,
                            content=_error_body("validation_error", "请求参数校验失败", exc.errors()))

    @app.exception_handler(Exception)
    async def handle_unexpected(request: Request, exc: Exception):
        logger.exception("unhandled error on %s %s", request.method, request.url.path)
        return JSONResponse(status_code=500,
                            content=_error_body("internal_error", "服务器内部错误", None))
```

注意：注册 `StarletteHTTPException` 处理器会覆盖 FastAPI 默认的 `{"detail": ...}` 输出，**前端凡是读取 `err.detail` 的地方都要在步骤 4 中排查**。

自查点：`python -c "from memory_anki.app.error_handlers import install_error_handlers"` 无报错。
