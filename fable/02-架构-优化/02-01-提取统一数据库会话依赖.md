# 02-01 提取统一数据库会话依赖

## 1. 原始需求

后端 11 个 presentation 路由文件各自复制了一份完全相同的 `session_dep()` 生成器（打开 session → yield → finally close），任何一处对会话生命周期的调整（如加事务包装、加只读标记）都要改 11 个地方。经 grep `def session_dep` 核实，11 处定义位于：

| 文件（相对 `apps/api/src/memory_anki/`） | 行号 |
|---|---|
| `modules/dashboard/presentation/router.py` | 13 |
| `modules/english/presentation/router.py` | 49 |
| `modules/english_reading/presentation/router.py` | 66 |
| `modules/freestyle/presentation/router.py` | 22 |
| `modules/knowledge/presentation/router.py` | 33 |
| `modules/palace_quiz/presentation/router.py` | 51 |
| `modules/palaces/presentation/import_router.py` | 56 |
| `modules/palaces/presentation/router.py` | 104 |
| `modules/reviews/presentation/router.py` | 42 |
| `modules/sessions/presentation/router.py` | 48 |
| `modules/settings/presentation/router.py` | 61 |

目标：新建一个公共依赖模块 `apps/api/src/memory_anki/infrastructure/db/deps.py`，11 个 router 全部改为从该模块导入 `session_dep`，行为完全不变。

**测试现状（撰写前已核实，执行者必须知晓）**：`apps/api/tests/` 共 26 个测试文件，其中 8 个通过给 router 模块的模块级属性赋值来替换 `get_session`（形如 `review_router.get_session = get_test_session`），合计约 42 处赋值，分布在：`test_study_session_routes.py`(4 处)、`test_review_routes.py`(20 处，含对 `main` 模块第 49 行 `get_session = _get_session` 别名的替换)、`test_palace_quiz_routes.py`(4)、`test_palace_chapter_binding.py`(4)、`test_english_routes.py`(4)、`test_mini_palace_routes.py`(2)、`test_freestyle_routes.py`(2)、`test_english_reading_routes.py`(2)。另有 `test_runtime_info.py` 第 243 行使用 `app.dependency_overrides[settings_router.session_dep]`。这些替换方式必须随本次改造同步迁移。

## 2. 详细执行清单

> 硬约束：本文档只允许改动上表 11 个 router 文件、新增 1 个 deps.py、以及 8 个测试文件中对 `get_session` 的替换语句。不要动 `tools/check_architecture.py`（11 个 router 都已在其 `BASELINE_PRESENTATION_SESSION_FILES` 基线内，不需要调整）；不要动 `apps/api/pyproject.toml`；不要"顺手"删除 router 里其他用途的 `get_session` 之外的任何 import。

### 步骤 1：新建公共依赖模块

新建文件 `apps/api/src/memory_anki/infrastructure/db/deps.py`，完整内容如下：

```python
"""FastAPI 路由共享的数据库会话依赖。

所有 presentation 路由统一从这里导入 ``session_dep``。
测试通过替换本模块的 ``get_session`` 属性注入测试会话工厂。
"""
from __future__ import annotations

from memory_anki.infrastructure.db.models import get_session


def session_dep():
    session = get_session()
    try:
        yield session
    finally:
        session.close()
```

注意：本文件不 import FastAPI（`session_dep` 是普通生成器函数，`Depends()` 在调用方使用），因此不违反任何分层约束。

自查点：`cd apps/api && python -c "from memory_anki.infrastructure.db.deps import session_dep; print(session_dep)"` 输出函数对象无报错。

### 步骤 2：替换 dashboard router（最小的一个，先验证模式）

打开 `apps/api/src/memory_anki/modules/dashboard/presentation/router.py`，找到第 13–18 行：

```python
def session_dep():
    session = get_session()
    try:
        yield session
    finally:
        session.close()
```

整段删除，并在文件顶部 import 区加入：

```python
from memory_anki.infrastructure.db.deps import session_dep
```

然后检查该文件是否还有其他地方直接调用 `get_session`（除被删除的 session_dep 外）。若没有，把 `from memory_anki.infrastructure.db.models import ...` 这行 import 里的 `get_session` 一项删掉；若 import 只剩 `get_session` 一项则整行删除。**若该文件其他函数还在用 `get_session`，保留 import 不动。**

同步迁移测试：`test_review_routes.py` 中所有 `dashboard_router.get_session = get_test_session` / `... = self.original_dashboard_get_session` 之类的赋值改为对 deps 模块赋值（见步骤 4 的统一写法）。

自查点：`cd apps/api && python -m pytest tests/test_review_routes.py -x -q` 全绿。

### 步骤 3：逐个替换其余 10 个 router

对上表其余 10 个文件重复步骤 2 的操作（删除本地 `def session_dep`，加 deps import，清理不再使用的 `get_session` import）。注意各文件局部变量名不同（有的写 `s = get_session()`，有的写 `session = get_session()`），行为相同，直接整段删除即可。每替换 1 个文件立即运行该模块对应的测试（见下面映射），不要一次性替换完再测试：

| router | 对应测试 |
|---|---|
| sessions | `tests/test_study_session_routes.py` |
| reviews / palaces / settings | `tests/test_review_routes.py` |
| palace_quiz | `tests/test_palace_quiz_routes.py` |
| knowledge | `tests/test_palace_chapter_binding.py` |
| freestyle | `tests/test_freestyle_routes.py` |
| english | `tests/test_english_routes.py` |
| english_reading | `tests/test_english_reading_routes.py` |
| palaces/import_router、dashboard | 无独立路由测试，跑全量 pytest 兜底 |
| palaces（mini palace 路由） | `tests/test_mini_palace_routes.py` |

自查点（每个文件替换后）：对应测试文件通过；`python -m ruff check src` 无新增 F401（未用 import）。

### 步骤 4：迁移测试对 get_session 的替换方式

改造后所有 router 共享 `deps.get_session`，测试只需替换一处。以 `tests/test_review_routes.py` 为例，setUp 中原来的：

```python
self.original_get_session = review_router.get_session
self.original_dashboard_get_session = dashboard_router.get_session
self.original_palace_get_session = palace_router.get_session
self.original_settings_get_session = settings_router.get_session
...
review_router.get_session = get_test_session
dashboard_router.get_session = get_test_session
palace_router.get_session = get_test_session
settings_router.get_session = get_test_session
```

改为：

```python
from memory_anki.infrastructure.db import deps as db_deps
...
self.original_get_session = db_deps.get_session
...
db_deps.get_session = get_test_session
```

tearDown 中对应恢复 `db_deps.get_session = self.original_get_session`。对 8 个测试文件逐一执行同样替换。注意事项：

- `test_review_routes.py` 中还有多处测试方法内替换 `main_module.get_session`（如 1486、1572、1781 行附近）：`app/main.py` 第 49 行的 `get_session = _get_session` 别名与路由无关（路由不读它），这些替换语句同样改为替换 `db_deps.get_session`，`main_module.get_session` 相关行删除。**不要在本文档中删除 `main.py` 第 49 行别名本身**（有独立文档处理 main.py 清理）。
- `test_runtime_info.py` 第 243 行 `app.dependency_overrides[settings_router.session_dep]` 不需要改：settings router `from ... import session_dep` 后模块属性依然存在且与 deps 中是同一函数对象，override 仍然命中。运行该测试确认即可。

自查点：`cd apps/api && python -m pytest -q` 全绿；`rg "\w+_router\.get_session\s*=" tests` 无结果。

### 步骤 5：全量回归

依次运行第 3 节全部命令。不要做的事：不要给 `session_dep` 加任何新行为（事务、日志等）；不要把 deps.py 放到 `memory_anki/app/` 下（presentation 反向依赖应用装配层不合理）；不要改 `infrastructure/db/models.py` 或 `_tables/_base.py`。
