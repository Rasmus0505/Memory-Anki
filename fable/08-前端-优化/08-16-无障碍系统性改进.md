# 08-16 无障碍系统性改进

## 1. 原始需求

前端主要无障碍风险集中在对话框缺标题/描述、keep-alive 页面仍可聚焦、命令面板缺可读名称。目标通过 shared dialog 默认约定、route inert 和命令面板隐藏描述建立基础保障。

## 2. 详细执行清单

1. `apps/web/src/shared/components/ui/dialog.tsx` 自动补隐藏 title/description 约定。
2. `apps/web/src/shared/components/ui/command.tsx` 为命令面板提供 accessible title/description。
3. `apps/web/src/app/router/AppRouter.tsx` 对非活跃 resident route 增加 `inert`。
4. 补 dialog、command、route residency 相关测试。

## 3. 测试验收标准

- `cd apps/web && npm run test -- src/shared/components/ui/dialog.test.tsx src/app/shell/GlobalCommandPalette.test.tsx src/app/router/AppRouter.test.tsx` 通过。
- `cd apps/web && npm run typecheck` 通过。
