# 08-13 统一加载态体系

## 1. 原始需求

多个页面曾维护各自 skeleton，视觉和文案不统一。目标是使用 shared loading/skeleton 组件和路由 fallback，减少重复加载态实现。

## 2. 详细执行清单

1. 使用 `apps/web/src/shared/components/ui/skeleton-layout.tsx` 与 `state-placeholders.tsx`。
2. `apps/web/src/app/router/appRoutes.tsx` 的 `RouteFallback` 使用统一 `LoadingState`。
3. 页面 skeleton 如 Dashboard、English、Review、Palace catalog 保持统一模式。

## 3. 测试验收标准

- `cd apps/web && npm run typecheck` 通过。
- `cd apps/web && npm run test -- src/app/router/appRoutes.fallback.test.ts src/shared/components/ui/dialog.test.tsx` 通过。
- 回归要求：路由 lazy fallback 仍能显示“正在加载页面…”。
