# 08-12 统一对话框使用规范

## 1. 原始需求

Radix Dialog 对缺失 title/description 会报警，调用点容易遗漏无障碍元信息。目标是在 shared dialog 层提供 `accessibleTitle` 与 `accessibleDescription` 约定。

## 2. 详细执行清单

1. 修改 `apps/web/src/shared/components/ui/dialog.tsx`。
2. `DialogContent` 支持 `accessibleTitle` / `accessibleDescription`，必要时自动渲染 sr-only 标题。
3. 无描述时默认关闭 optional description warning。
4. 修改 `apps/web/src/shared/components/ui/command.tsx`，为命令面板声明隐藏标题/描述。
5. 补 `apps/web/src/shared/components/ui/dialog.test.tsx`。

## 3. 测试验收标准

- `cd apps/web && npm run test -- src/shared/components/ui/dialog.test.tsx src/app/shell/GlobalCommandPalette.test.tsx` 通过。
- 输出中不再出现 DialogTitle/DialogDescription 警告。
- `cd apps/web && npm run typecheck` 通过。
