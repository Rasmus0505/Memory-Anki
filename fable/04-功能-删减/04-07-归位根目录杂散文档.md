# 04-07 归位根目录杂散文档

## 1. 原始需求

对仓库根目录逐一盘点（2026-07-08，`Get-ChildItem D:\322321\Memory-Anki`）后，真正的杂散文件只有一个：

- `代码优化提示词.md`（119 行）——一份通用的"AI 代码优化审查提示词模板"，内容与 Memory Anki 项目本身无关（甚至示例里引用的 `apps/web/src/features/mobile` 目录已不存在），属于个人提示词素材，混在项目根目录。

根目录全量盘点表（除目录外的所有文件，逐一给出处置结论）：

| 文件 | 作用 | 处置 |
|---|---|---|
| `AGENT.md` | 硬性协作规则（998 字节） | 保留，禁止移动 |
| `AI_PROJECT_CONTEXT.md` | AI 主上下文（约 16KB） | 保留，仅修一行描述（见步骤 2） |
| `PWA.md` | PWA 唯一现行说明 | 保留，禁止移动 |
| `代码优化提示词.md` | 通用 AI 提示词模板，与项目无关 | **删除（本文档唯一删减对象）** |

| `start-pwa.bat` / `stop-pwa.bat` | PWA 启动/停止 | 保留 |

| `.editorconfig` / `.gitattributes` / `.gitignore` / `.env.example` | 工程配置 | 保留 |

结论：所谓"等杂散文件"经核实实际只有 `代码优化提示词.md` 一个（与任务描述的复数假设不符，已在文末进度表注明）。

关于"移入 docx/"的核实结果（与任务假设不符，重要）：**`docx/` 目录在本仓库中不存在**（`Test-Path D:\322321\Memory-Anki\docx` 为 False，且 `.gitignore` 中没有 docx 条目）。`AI_PROJECT_CONTEXT.md` 第 94 行的目录树把 `docx/` 画进了仓库结构里，但它实际位于百度网盘同步盘——证据是 `tools/organize_memory_anki_docs.py` 第 8 行 `ROOT = Path(r"D:\BaiduSyncdisk\Memory Anki\docx\Memory Anki")`。因此"移入 docx/"不是一个 git 内可执行的归位动作。

决策：
1. `代码优化提示词.md` → **git rm 删除**。它是通用素材而非项目文档，git 历史可随时找回；若用户想继续使用，应放进百度网盘的个人笔记区，而不是项目仓库（执行代理无法写网盘路径，删除前在进度表留一句提醒即可）。
2. `AI_PROJECT_CONTEXT.md` 第 94 行 → **补一处澄清**，注明 docx/ 不在 git 仓库内，防止后续代理去仓库里找一个不存在的目录。
