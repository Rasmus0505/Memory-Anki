# 04-04 归档一次性 1000 题库工具链

## 1. 原始需求

`tools/` 下有 11 个"1000 题库"数据处理脚本，是一条一次性流水线（对考研 1000 题 PDF 做 OCR →
定位题目页 → 生成清单 → 审计题库 → 修复/回灌），合计约 20 万字节，其中
`repair_1000_quiz_bank_local.py` 一个文件就有 105 KB。它们与 `dev_server.py`、`check_architecture.py`
等日常工具混在同一层，且多个脚本硬编码个人绝对路径（如 `ocr_1000_questions.py` 第 15 行
`D:\考研（丹丹）\1000题`、`audit_1000_quiz_bank.py` 第 17 行 `D:\BaiduSyncdisk\MemoryAnki-Sync`），
`python tools/check_architecture.py` 当前已经在报 `tools/backfill_1000_quiz_ocr_sources.py` 与
`tools/repair_1000_quiz_bank_local.py` 两条个人路径违规（属既有失败项）。

**运行时引用面核实结论**（`rg` 全仓检索 11 个模块名）：`apps/`（前后端运行时代码）零引用。仅有三处
tools 内部引用：

1. `tools/repair_1000_quiz_bank_local.py` 第 1720-1721 行按文件路径动态加载
   `tools/audit_1000_quiz_bank.py`；
2. `tools/backfill_1000_quiz_ocr_sources.py` 第 17 行 `from repair_1000_quiz_bank_local import ...`
   （同目录 import，脚本一起移动后依然成立）；
3. `tools/check_architecture.py` 第 127-136 行 `BASELINE_PERSONAL_PATH_TOOLS` 豁免清单中列了其中 7 个
   脚本的路径字符串。

目标：11 个脚本整体移入 `tools/archive/1000-quiz/`，修正随之失效的路径推导与引用，补一份 README
说明用途与复跑方法，并同步 check_architecture 豁免清单（顺带让上面两条既有报错消失）。

**要移动的 11 个脚本**：`ocr_1000_questions.py`、`locate_1000_quiz_pages.py`、
`render_1000_question_pages.py`、`merge_1000_page_drafts.py`、`build_1000_questions_manifest.py`、
`audit_1000_quiz_bank.py`、`repair_1000_quiz_bank_local.py`、`backfill_1000_quiz_ocr_sources.py`、
`build_1000_quiz_rerun_plan.py`、`run_1000_quiz_rerun_plan.py`、`save_1000_quiz_preview_results.py`。

### 步骤 3：修正移动后失效的 REPO_ROOT 推导

`tools/archive/1000-quiz/` 后（多了两层）必须改成 `parents[3]`：

| 文件 | 原行号 |
|---|---|
| `audit_1000_quiz_bank.py` | 16 |
| `build_1000_quiz_rerun_plan.py` | 13 |
| `locate_1000_quiz_pages.py` | 15 |
| `run_1000_quiz_rerun_plan.py` | 14 |
| `save_1000_quiz_preview_results.py` | 11 |
| `repair_1000_quiz_bank_local.py` | 23 |
| `backfill_1000_quiz_ocr_sources.py` | 12 |

每处修改前/后：

```python
# 修改前
REPO_ROOT = Path(__file__).resolve().parents[1]
# 修改后（脚本已归档到 tools/archive/1000-quiz/，需回退三层到仓库根）
REPO_ROOT = Path(__file__).resolve().parents[3]
```

另外 4 个脚本（`ocr_1000_questions.py`、`render_1000_question_pages.py`、`merge_1000_page_drafts.py`、
`build_1000_questions_manifest.py`）只用硬编码绝对路径，没有 REPO_ROOT，不用改。

- **自查点**：`rg -n "parents\[1\]" tools/archive/1000-quiz` 为空。

### 步骤 6：撰写归档 README

新建 `tools/archive/1000-quiz/README.md`，内容要点（用中文写全）：

```markdown
# 1000 题库一次性处理流水线（已归档）

本目录是 2026 年上半年针对考研《1000 题》PDF 题库的一次性数据处理脚本，日常开发不需要运行。

## 各脚本用途（按流水线顺序）
1. ocr_1000_questions.py —— 对题册/解析册 PDF 逐页 OCR（依赖 rapidocr_onnxruntime、PyMuPDF）
2. render_1000_question_pages.py —— 把 PDF 页渲染成图片
3. merge_1000_page_drafts.py —— 合并人工校对的分页草稿
4. build_1000_questions_manifest.py —— 生成章节/题目清单
5. audit_1000_quiz_bank.py —— 审计库内 1000 题库快照的数据质量
6. build_1000_quiz_rerun_plan.py —— 依据审计结果生成重跑计划
7. locate_1000_quiz_pages.py —— 为重跑计划定位 PDF 页码
8. run_1000_quiz_rerun_plan.py —— 执行重跑计划（调用 apps/api 的出题流程）
9. save_1000_quiz_preview_results.py —— 保存预览结果
10. repair_1000_quiz_bank_local.py —— 本地修复题库（内部动态加载 audit 脚本）
11. backfill_1000_quiz_ocr_sources.py —— 把 OCR 来源回灌到数据库（import repair 脚本）

## 复跑方法
- 在仓库根目录执行：`python tools/archive/1000-quiz/<脚本名>.py`（带 REPO_ROOT 的脚本已改为 parents[3]，
  可在任意 cwd 运行）。
- 注意：多个脚本硬编码了个人素材/网盘绝对路径（D:\考研（丹丹）、D:\BaiduSyncdisk、D:\我的网站），
  换机器复跑前必须先改这些常量；中间产物默认写到仓库根的 .audit/ 目录（已被 gitignore）。
- 运行时代码（apps/）不依赖本目录，删除本目录不影响应用。
```
