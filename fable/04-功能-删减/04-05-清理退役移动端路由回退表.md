# 04-05 清理退役移动端路由回退表

## 1. 原始需求

`apps/web/src/app/router/appRoutes.tsx` 第 118-127 行有一张退役移动端路由回退表：

```118:127:apps/web/src/app/router/appRoutes.tsx
const RETIRED_MOBILE_ROUTE_FALLBACKS: Record<string, string> = {
  '/m': '/freestyle',
  '/mobile': '/freestyle',
  '/m/freestyle': '/freestyle',
  '/mobile/freestyle': '/freestyle',
  '/m/palaces': '/palaces',
  '/mobile/palaces': '/palaces',
  '/m/mindmap': '/palaces',
  '/mobile/mindmap': '/palaces',
}
```

它在 `resolveRouteFallbackTarget`（第 132-133 行）中最先被查询，服务于独立移动端 `/m` 路由退役时
（AI_PROJECT_CONTEXT.md 第 12 节记载：2026-07-06 起 PWA 恢复为完整桌面端入口，默认 `/freestyle`）
的旧书签/PWA 缓存跳转。

**保留价值评估**：

- 全仓 `rg` 核实，站内代码没有任何链接指向 `/m` 或 `/mobile`（仅此表与其测试引用这些字符串）。
- 表中 8 条映射有 4 条（`/m`、`/mobile`、`/m/freestyle`、`/mobile/freestyle`）的目标与
  `resolveRouteFallbackTarget` 的兜底默认值 `'/freestyle'`（第 149 行）**完全相同**——删表后行为不变。
- 剩下 4 条（`/m|mobile/palaces`、`/m|mobile/mindmap` → `/palaces`）删表后会改为回退 `/freestyle`。
  影响仅限"手机上还留着指向旧移动端宫殿页的书签"这一场景；本产品仅两台自有设备 + 手机 PWA，且
  PWA 已按 PWA.md 重新配置为桌面端入口，落到 `/freestyle`（当前默认主页）后一次点击即可到宫殿页。

**结论：采用"立即精简"方案**——整表删除，不再等待观察期。若执行时用户手机上确有 `/m/palaces` 旧书签，
删除后的行为退化（进 `/freestyle` 而非 `/palaces`）是可接受的，且可通过更新书签一次性解决。

```typescript
export function resolveRouteFallbackTarget(pathname: string) {
  const normalizedPathname = normalizePathname(pathname)

  if (REGISTERED_EXACT_PATHS.has(normalizedPathname)) return normalizedPathname
```

修改后：

```typescript

  const normalizedPathname = normalizePathname(pathname)

  if (REGISTERED_EXACT_PATHS.has(normalizedPathname)) return normalizedPathname
```
