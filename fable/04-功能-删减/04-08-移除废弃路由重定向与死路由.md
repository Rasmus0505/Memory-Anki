# 04-08 移除废弃路由重定向与死路由

## 1. 原始需求

`apps/web/src/app/router/appRoutes.tsx` 中有三条历史重定向路由（行号已核实，2026-07-08）：

- 第 171 行：`<Route path="/palaces/quiz" element={<Navigate to="/palaces" replace />} />`
- 第 187 行：`<Route path="/profile/ai-prompts" element={<Navigate to="/profile/ai?tab=prompts" replace />} />`
- 第 188 行：`<Route path="/profile/ai-split" element={<Navigate to="/profile/ai?tab=config" replace />} />`

全前端 grep 核实站内引用：没有任何 `<Link>` / `navigate()` 指向这三个旧路径。仅有的三处字符串出现均不构成依赖——`appRoutes.fallback.test.ts` 第 22 行测试的是 `resolveRouteFallbackTarget('/palaces/quiz')` 纯函数（该函数只服务于通配符 404 路由，删不删 `<Route>` 都不影响它）；`ProfileAiPromptsPage.test.tsx` 第 51 行只是 MemoryRouter 的 initialEntries，不经过 AppRoutes。

与任务假设不符之处（两条，均影响执行方案）：

1. **`/palaces/quiz` 不能删。** 删掉第 171 行后，该路径不会落入通配符回退，而是被上一行注册的动态路由 `/palaces/:id`（第 172 行）捕获，`PalaceViewPage` 里 `Number('quiz')` 得到 NaN（PalaceViewPage.tsx 第 32 行），触发对 `/palaces/NaN` 的编辑器请求并渲染错误态。这条重定向是挡在动态路由前面的"防弹片"，必须保留，本文档为它补注释防止将来被误删。
2. **`REGISTERED_EXACT_PATHS` 无需同步。** 核实第 68–85 行集合内容，三个旧路径本来就不在其中，没有要删的条目。fallback 测试同理无需修改（第 22 行用例在删除后依然成立且必须继续通过）。

因此实际删减对象为第 187、188 两行。三条路径删除前后的行为对照（已按 appRoutes.tsx 的匹配顺序逐条推演）：

| 旧路径 | 现行为 | 删 `<Route>` 后的行为 | 结论 |
|---|---|---|---|
| `/profile/ai-prompts` | 302 到 `/profile/ai?tab=prompts` | 通配符 404 → `SECTION_PREFIX_FALLBACKS['/profile/']` → `/profile` | 可删，降级可接受 |
| `/profile/ai-split` | 302 到 `/profile/ai?tab=config` | 同上 → `/profile` | 可删，降级可接受 |
| `/palaces/quiz` | 302 到 `/palaces` | 被 `/palaces/:id` 捕获，`id='quiz'` → NaN → 请求 `/palaces/NaN` 报错页 | **不可删，保留+注释** |

书签兼容评估：个人自用产品、双设备，旧路径只可能存在于浏览器历史或 PWA 快捷方式里；落到 `/profile` 后手动点一次"AI 设置"即可，成本可接受。

### 步骤 1：删除两条 profile 旧重定向

打开 `apps/web/src/app/router/appRoutes.tsx`，在 `<Routes>` 块中找到并整行删除第 187、188 行：

```tsx
// 修改前（183–189 行）
<Route path="/profile" element={<ProfilePage />} />
<Route path="/profile/timer" element={<ProfileTimerPage />} />
<Route path="/profile/feedback" element={<ProfileFeedbackPage />} />
<Route path="/profile/ai" element={<ProfileAiPage />} />
<Route path="/profile/ai-prompts" element={<Navigate to="/profile/ai?tab=prompts" replace />} />
<Route path="/profile/ai-split" element={<Navigate to="/profile/ai?tab=config" replace />} />
<Route path="/profile/backups" element={<ProfileBackupsPage />} />

// 修改后
<Route path="/profile" element={<ProfilePage />} />
<Route path="/profile/timer" element={<ProfileTimerPage />} />
<Route path="/profile/feedback" element={<ProfileFeedbackPage />} />
<Route path="/profile/ai" element={<ProfileAiPage />} />
<Route path="/profile/backups" element={<ProfileBackupsPage />} />
```

不要做什么：
- 不要删 `/profile/ai` 本体路由与 `ProfileAiPage` 的 lazy import（第 49 行）——只有重定向是废弃的，目标页是现役的。
- 不要删 `ProfileAiPromptsPage.test.tsx`，也不要改它的 initialEntries——该测试直接渲染组件，与路由表无关。

自查点：`rg "ai-prompts|ai-split" apps/web/src/app/router/appRoutes.tsx` 输出为空。

### 步骤 2：为 /palaces/quiz 重定向补防误删注释

同文件第 171 行，在该行上方加一行注释（不改路由本身）：

```tsx
// 修改前
<Route path="/palaces/quiz" element={<Navigate to="/palaces" replace />} />
<Route path="/palaces/:id" element={<PalaceViewPage />} />

// 修改后
{/* 保留：若删除此行，/palaces/quiz 会被下面的 /palaces/:id 捕获，Number('quiz')=NaN 导致坏页。 */}
<Route path="/palaces/quiz" element={<Navigate to="/palaces" replace />} />
<Route path="/palaces/:id" element={<PalaceViewPage />} />
```

不要做什么：不要"顺手"把 `/palaces/quiz` 挪到 `REGISTERED_EXACT_PATHS` 或改造 PalaceViewPage 的 NaN 防御——超出本文档最小改动范围。

自查点：第 171 行路由原样保留，仅多一行注释。

### 步骤 3：确认无需同步的部分（只核对，不改动）

任务原始假设是"删除后需同步 REGISTERED_EXACT_PATHS 与 fallback 测试"，核实结果均为**无需改动**，逐项说明如下，执行者只做核对：
