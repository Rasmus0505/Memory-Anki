# 06-01 全局搜索

## 1. 原始需求

目前应用没有任何跨模块搜索能力：

1. `apps/web/src/app/shell/GlobalCommandPalette.tsx`（129 行）已实现 Ctrl+K 呼出的 cmdk 命令面板，但只有 6 个写死的导航动作（开始复习/新建宫殿/搜索宫殿等），其中"搜索宫殿"只是跳转到 `/palaces/list?focusSearch=true` 做前端列表过滤。
2. 记忆桩内容（`pegs` 表的 `name`/`content` 列，见 `apps/api/src/memory_anki/infrastructure/db/_tables/palaces.py` 第 151-182 行）、题目题干（`palace_quiz_questions.stem`，同文件第 461 行）、知识章节（`chapters.name`，`_tables/knowledge.py` 第 31-67 行）完全没有搜索入口。想找"当年在哪个宫殿记过某个知识点"只能逐个打开宫殿翻。

期望效果：Ctrl+K 面板输入 ≥2 个字符时，实时展示跨宫殿标题 / 记忆桩 / 题目 / 章节四组搜索结果，回车直接跳到对应页面。

**FTS5 vs LIKE 取舍（本文档结论：LIKE 起步）**：个人单用户数据量级为几百个宫殿、几千条 pegs/题目，SQLite `LIKE '%q%'` 全表扫描在该量级下毫秒级返回；FTS5 需要建虚拟表 + 三个同步触发器 + 一次全量重建迁移，且对中文需要额外分词方案（默认 unicode61 按字切分对中文 LIKE 场景无优势）。按 1.md 克制原则，先用 LIKE 实现，接口形状保持稳定，未来数据量出问题时再单独提 FTS5 迁移文档，前端无需改动。

```python
"""Global search across palaces, pegs, quiz questions and chapters (LIKE-based)."""

from __future__ import annotations

from sqlalchemy import or_
from sqlalchemy.orm import Session, joinedload

from memory_anki.infrastructure.db.models import (
    Chapter,
    Palace,
    PalaceQuizQuestion,
    Peg,
)
from memory_anki.modules.palaces.application.title_sync_service import resolve_palace_title

PER_GROUP_LIMIT = 10
_LIKE_ESCAPE = "\\"


def _like_pattern(query: str) -> str:
    escaped = (
        query.replace(_LIKE_ESCAPE, _LIKE_ESCAPE + _LIKE_ESCAPE)
        .replace("%", _LIKE_ESCAPE + "%")
        .replace("_", _LIKE_ESCAPE + "_")
    )
    return f"%{escaped}%"


def _snippet(text: str, query: str, radius: int = 30) -> str:
    content = (text or "").strip()
    if not content:
        return ""
    index = content.lower().find(query.lower())
    if index < 0:
        return content[: radius * 2]
    start = max(0, index - radius)
    end = min(len(content), index + len(query) + radius)
    prefix = "…" if start > 0 else ""
    suffix = "…" if end < len(content) else ""
    return f"{prefix}{content[start:end]}{suffix}"


def global_search(session: Session, query: str, limit: int = PER_GROUP_LIMIT) -> dict:
    query = (query or "").strip()
    if len(query) < 1:
        return {"query": query, "palaces": [], "pegs": [], "questions": [], "chapters": []}
    limit = max(1, min(int(limit), 20))
    pattern = _like_pattern(query)

    palaces = (
        session.query(Palace)
        .filter(
            Palace.archived == False,  # noqa: E712
            or_(
                Palace.title.like(pattern, escape=_LIKE_ESCAPE),
                Palace.manual_title.like(pattern, escape=_LIKE_ESCAPE),
                Palace.description.like(pattern, escape=_LIKE_ESCAPE),
            ),
        )
        .order_by(Palace.updated_at.desc())
        .limit(limit)
        .all()
    )

    pegs = (
        session.query(Peg)
        .options(joinedload(Peg.palace))
        .join(Palace, Peg.palace_id == Palace.id)
        .filter(
            Palace.archived == False,  # noqa: E712
            or_(
                Peg.name.like(pattern, escape=_LIKE_ESCAPE),
                Peg.content.like(pattern, escape=_LIKE_ESCAPE),
            ),
        )
        .limit(limit)
        .all()
    )

    questions = (
        session.query(PalaceQuizQuestion)
        .options(joinedload(PalaceQuizQuestion.palace))
        .filter(PalaceQuizQuestion.stem.like(pattern, escape=_LIKE_ESCAPE))
        .order_by(PalaceQuizQuestion.updated_at.desc())
        .limit(limit)
        .all()
    )

    chapters = (
        session.query(Chapter)
        .options(joinedload(Chapter.subject))
        .filter(Chapter.name.like(pattern, escape=_LIKE_ESCAPE))
        .limit(limit)
        .all()
    )

    return {
        "query": query,
        "palaces": [
            {
                "id": palace.id,
                "title": resolve_palace_title(palace),
                "snippet": _snippet(palace.description, query),
            }
            for palace in palaces
        ],
        "pegs": [
            {
                "id": peg.id,
                "palace_id": peg.palace_id,
                "palace_title": resolve_palace_title(peg.palace) if peg.palace else "",
                "name": peg.name,
                "snippet": _snippet(peg.content or peg.name, query),
            }
            for peg in pegs
        ],
        "questions": [
            {
                "id": question.id,
                "palace_id": question.palace_id,
                "palace_title": resolve_palace_title(question.palace) if question.palace else "",
                "snippet": _snippet(question.stem, query),
            }
            for question in questions
        ],
        "chapters": [
            {
                "id": chapter.id,
                "name": chapter.name,
                "subject_name": chapter.subject.name if chapter.subject else "",
            }
            for chapter in chapters
        ],
    }
```

说明：`resolve_palace_title` 已存在于 `apps/api/src/memory_anki/modules/palaces/application/title_sync_service.py`（`freestyle/application/feed_service.py` 第 21 行同样这么引用），负责处理 `title_mode=manual` 时用 `manual_title` 的规则，**不要**自己拼标题。

from fastapi import APIRouter, Depends, Query
from sqlalchemy.orm import Session

from memory_anki.infrastructure.db.models import get_session
from memory_anki.modules.search.application.search_service import global_search

router = APIRouter(tags=["search"])


def session_dep():
    session = get_session()
    try:
        yield session
    finally:
        session.close()


@router.get("/search")
def api_global_search(
    q: str = Query(default=""),
    limit: int = Query(default=10),
    session: Session = Depends(session_dep),
):
    return global_search(session, q, limit)

#### 步骤 4：GlobalCommandPalette 接入远程搜索

打开 `apps/web/src/app/shell/GlobalCommandPalette.tsx`（129 行），做以下修改：

1. 新增受控输入与防抖查询状态：

```tsx
import { useDeferredValue } from 'react'
import { globalSearchApi } from '@/entities/search/api'
import type { GlobalSearchResponse } from '@/shared/api/contracts'

const [query, setQuery] = useState('')
const [results, setResults] = useState<GlobalSearchResponse | null>(null)
const deferredQuery = useDeferredValue(query)

useEffect(() => {
  const trimmed = deferredQuery.trim()
  if (trimmed.length < 2) {
    setResults(null)
    return
  }
  let cancelled = false
  const timer = window.setTimeout(() => {
    globalSearchApi(trimmed)
      .then((data) => {
        if (!cancelled) setResults(data)
      })
      .catch(() => {
        if (!cancelled) setResults(null)
      })
  }, 200)
  return () => {
    cancelled = true
    window.clearTimeout(timer)
  }
}, [deferredQuery])
```

2. `CommandInput` 改为受控：`<CommandInput value={query} onValueChange={setQuery} placeholder="搜索操作、宫殿、记忆桩、题目、章节..." />`。
3. `CommandDialog` 打开状态变化时清空：在现有 `useEffect(() => { setOpen(false) }, [location...])` 附近加 `useEffect(() => { if (!open) { setQuery(''); setResults(null) } }, [open])`。
4. 远程结果分组渲染。在现有 `<CommandGroup heading="常用操作">` 之后追加四个条件分组（有结果才渲染）。**关键**：cmdk 默认会按输入再过滤一次 item，远程结果的展示文本可能与查询词不完全匹配而被 cmdk 滤掉，因此每个远程 `CommandItem` 的 `value` 必须拼上当前查询词保证命中，例如 `value={\`palace-${hit.id}-${query}\`}`：

```tsx
{results && results.palaces.length > 0 && (

    {results.palaces.map((hit) => (
      <CommandItem
        key={`palace-${hit.id}`}

          navigate(`/palaces/${hit.id}`)

```
