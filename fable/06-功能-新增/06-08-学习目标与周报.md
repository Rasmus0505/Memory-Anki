# 06-08 学习目标与周报

## 1. 原始需求

现状：dashboard（`apps/web/src/features/dashboard/DashboardPage.tsx`）已显示本周复习时长（后端 `build_dashboard_payload` 的 `weekly_total_review_duration_seconds` 等字段，见 `modules/dashboard/application/service.py` 第 105~156 行），但没有"目标"概念——看不出本周学得够不够；也没有上周回顾。

期望：
1. 目标设置：每周学习时长目标（分钟）与每周复习完成次数目标，存进既有 client-preferences 机制（`GET/PUT /api/v1/profile/client-preferences`，`modules/settings/presentation/router.py` 第 200~207 行；分组白名单 `CLIENT_PREFERENCE_GROUPS` 在同文件第 46~56 行，值以 JSON 存 `config` 表的 `client_preferences.<group>` key）。
2. 周报：后端新增 `GET /api/v1/dashboard/weekly-report`，纯查询计算上一自然周（周一~周日）的学习时长、复习次数、平均分、新建宫殿数——打开页面时算，不需要定时任务。
3. dashboard 新增"本周目标"卡片（进度条 + 编辑目标）与"上周摘要"卡片。

已核实的可复用函数（`modules/sessions/application/study_session_service.py`，dashboard service 第 22~34 行已 import）：`current_week_bounds()`、`get_study_session_duration_seconds(session, scenes=..., start=..., end=...)`、常量 `STUDY_DASHBOARD_SCENES`；`review_logs` 表有 `review_date/score`。

本期不做：
- 不做目标提醒/通知（无推送渠道）；
- 不做周报历史归档（每次现算，只看上一周）；
- 不做按科目拆分的目标；
- 不引入定时任务。

2. 在 `apps/api/src/memory_anki/modules/dashboard/application/service.py` 末尾追加周报函数：

```python
def build_weekly_report_payload(session: Session, *, offset_weeks: int = 1) -> dict:
    """计算某一自然周（默认上一周）的学习摘要。offset_weeks=0 表示本周。"""
    from memory_anki.infrastructure.db.models import ReviewLog

    safe_offset = max(0, min(int(offset_weeks or 0), 52))
    current_week_start, _current_week_end = current_week_bounds()
    week_start = current_week_start - timedelta(days=7 * safe_offset)
    week_end = week_start + timedelta(days=7)

    study_seconds = get_study_session_duration_seconds(
        session,
        scenes=STUDY_DASHBOARD_SCENES,
        start=week_start,
        end=week_end,
    )
    review_rows = (
        session.query(ReviewLog)
        .filter(
            ReviewLog.review_date >= week_start.date(),
            ReviewLog.review_date < week_end.date(),
        )
        .all()
    )
    review_count = len(review_rows)
    average_score = (
        round(sum(int(row.score or 0) for row in review_rows) / review_count, 1)
        if review_count
        else 0
    )
    new_palace_count = (
        session.query(func.count(Palace.id))
        .filter(
            Palace.created_at.is_not(None),
            Palace.created_at >= week_start,
            Palace.created_at < week_end,
        )
        .scalar()
        or 0
    )
    return {
        "week_start": week_start.date().isoformat(),
        "week_end": (week_end - timedelta(days=1)).date().isoformat(),
        "study_seconds": int(study_seconds or 0),
        "review_count": review_count,
        "average_score": average_score,
        "new_palace_count": int(new_palace_count),
    }
```

   - 文件顶部已 import `timedelta/func/Palace/current_week_bounds/get_study_session_duration_seconds/STUDY_DASHBOARD_SCENES`（第 3~34 行），`ReviewLog` 用函数内局部 import。
   - 不要改 `build_dashboard_payload`。
   - 自查点：`python -c "from memory_anki.modules.dashboard.application.service import build_weekly_report_payload"` 无报错。

3. 在 `apps/api/src/memory_anki/modules/dashboard/presentation/router.py` 注册路由（import 区补 `build_weekly_report_payload`）：

@router.get("/dashboard/weekly-report")
def api_weekly_report(

```ts
export interface StudyGoals {
  weekly_study_minutes: number
  weekly_review_count: number
}

export const DEFAULT_STUDY_GOALS: StudyGoals = {
  weekly_study_minutes: 300,
  weekly_review_count: 20,
}

export function getStudyGoalsApi() {
  return request<{ items: Record<string, unknown> }>(`/profile/client-preferences`).then(
    (response) => (response.items?.study_goals as StudyGoals | null) ?? null,
  )
}

export function saveStudyGoalsApi(goals: StudyGoals) {
  return request<{ items: Record<string, unknown> }>(`/profile/client-preferences`, {
    method: 'PUT',
    body: JSON.stringify({ study_goals: goals }),
    persistence: {
      resourceKey: 'preferences:study-goals',
      coalesceKey: 'preferences:study-goals',
      description: '保存学习目标',
      replayMode: 'auto',
    },
  })
}
```

   放进第 4 步同一个 API 文件。
   - 自查点：PUT 后 GET 能取回同一对象。

## 3. 测试验收标准

可执行命令：
- `curl "http://127.0.0.1:8012/api/v1/dashboard/weekly-report?offset_weeks=1"` → 返回上周日期区间的 JSON。
- `curl "http://127.0.0.1:8012/api/v1/dashboard/weekly-report?offset_weeks=0"` → `week_start` 为本周一。
- client-preferences 写读往返（见步骤 1 自查点命令）。
- `cd apps/web && npm run typecheck && npm run test` → 通过。

行为验收（操作 → 期望现象）：
1. 打开 dashboard → 出现"本周目标"卡片（默认 300 分钟 / 20 次）与"上周摘要"卡片。
2. 点"编辑"把时长目标改为 60 分钟 → 进度条比例立即变化；刷新页面后目标仍是 60（服务端已持久化）。
3. 在另一台设备（或清 localStorage 后）打开 dashboard → 目标同样是 60 分钟（跨设备一致）。
4. 本周完成复习使次数达标 → 对应进度条到 100% 并显示"已达成"。
5. 全新空库 → 上周摘要显示"上周没有学习记录"，目标卡片用默认值，不报错。

回归检查：
- `PUT /profile/client-preferences` 不影响其他 group（快捷键、计时器等设置原样保留——PUT 只更新提交的 group）。
- `GET /api/v1/dashboard` 响应不变。
- dashboard 既有"本周复习时长"统计与新目标卡片数值同源不冲突（都来自 study_sessions）。
