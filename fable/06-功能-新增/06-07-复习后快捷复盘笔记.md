# 06-07 复习后快捷复盘笔记

## 1. 原始需求

现状：复习会话结束时只记录分数与时长——`review_logs` 表（`apps/api/src/memory_anki/infrastructure/db/_tables/palaces.py` 第 295~312 行）仅有 `palace_id/review_date/score/review_mode/duration_seconds`。哪个知识点卡壳、下次要注意什么，全靠脑子记。提交链路已核实：前端 `apps/web/src/features/review/ReviewSessionContainer.tsx` 手动完成时先弹 `StageSelectDialog`（`features/review/components/StageSelectDialog.tsx`，`onConfirm(targetReviewNumber, needsPractice)`），随后 `handleStageConfirm`（第 331~350 行）调 `submitReviewSessionApi` → `POST /api/v1/review/session/{id}/submit`；后端路由 `modules/reviews/presentation/router.py` 第 180~220 行接收 `data: dict`，`submit_review(..., commit=False)` 返回 `log`（ReviewLog 实例）后由路由统一 `session.commit()`。

期望：`review_logs` 加 `note` 列（alembic 迁移）；提交 payload 增加可选 `note` 字段，路由把它写到当次 log 上；`StageSelectDialog` 增加一个可选的复盘输入框（"这次哪里卡了？"）；dashboard 新增"最近复盘"卡片回看最近 N 条带笔记的复习记录。

本期不做：
- 自动完成（auto_complete，不弹阶段选择弹窗）路径不加笔记入口；
- 不做笔记编辑/删除（写错就下次复习重写一条）；
- 不做按宫殿聚合的笔记页，只在 dashboard 回看最近若干条；
- segment/mini-palace 复习的日志表不在本期加列。

```python
def list_recent_review_notes(session, limit: int = 20) -> list[dict]:
    safe_limit = max(1, min(int(limit or 20), 100))
    rows = (
        session.query(ReviewLog, Palace)
        .join(Palace, Palace.id == ReviewLog.palace_id)
        .filter(ReviewLog.note != "")
        .order_by(ReviewLog.id.desc())
        .limit(safe_limit)
        .all()
    )
    return [
        {
            "id": log.id,
            "palace_id": log.palace_id,
            "palace_title": palace.manual_title or palace.title,
            "review_date": log.review_date.isoformat() if log.review_date else None,
            "note": log.note,
        }
        for log, palace in rows
    ]
```
