# 05-13 AI 模型注册表与场景映射管理体验

## 1. 原始需求

AI 模型配置需要支持 provider/category/scenario 维度管理，避免不同功能散落硬编码模型。目标是通过 settings AI registry 提供统一场景映射与影响分析。

## 2. 详细执行清单

1. 使用 `apps/api/src/memory_anki/modules/settings/application/ai_model_registry_*` 维护模型 catalog、scenario runtime 与 insights。
2. `apps/api/src/memory_anki/modules/settings/presentation/router.py` 提供场景更新、catalog upsert、impact、test、delete 接口。
3. 在 `apps/api/tests/test_settings_ai_routes.py` 覆盖列表、保存共享模型、场景 override、影响分析与删除错误。

## 3. 测试验收标准

- `cd apps/api && python -m pytest tests/test_settings_ai_routes.py -q` 通过。
- 回归要求：旧的 `scenario_updates` 请求仍可兼容。
