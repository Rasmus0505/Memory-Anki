# 05-05 思维导图导入 job 错误恢复与断点续跑

## 1. 原始需求

思维导图导入（`apps/api/src/memory_anki/modules/palaces/application/mindmap_import/`，24 个文件）的断点续跑地基**已经存在但没接通**：

1. **阶段级断点已实现**。`job_worker.py` 里每个阶段的产物都落盘并在重跑时跳过：单图 OCR 文本存 `extracted_text.txt`（第 78 行判存在则跳过）、结构树存 `structure_tree.json`（第 128 行）、最终结果存 `result.json`（第 175、231 行）。每完成一个阶段就 `set_job_stage` + `update_job_usage` 提交（第 108-109、167-168 行）。也就是说：失败/暂停后重新 `POST /import/jobs/{id}/run`，已完成的 LLM 调用**不会重复计费**。
2. **但服务重启会把 job 永远卡在 `running`**。任务在本地守护线程执行（`job_lifecycle.py` 的 `run_job_async` 第 33-51 行），重启后线程消失、DB 里状态仍是 `running`。`job_state.py` 第 10 行定义了 `JOB_STATUS_INTERRUPTED = "interrupted"`，`serialize_job` 也把它算进 `resumable`（`job_repository.py` 第 151-152 行），**可是全仓没有任何代码把状态置为 `interrupted`**（已用 `rg` 确认只有定义与转发）。后果：前端 `useImportJobRuntime.ts`（第 100、206、238、257 行）见到 `running` 就无限轮询，抽屉里转圈永不结束，"继续识别"按钮也不出现。
3. **失败明细已结构化但前端只显示一句话**。`mark_job_failed`（`job_repository.py` 第 311-340 行）会写入结构化错误（`job_errors.py` 第 9-49 行：`code`/`stage`/`message`/`retryable`/`raw_snippet`/`request_id`），并按关键词分类出 `provider_network_error`（含 LLM 超时 timed out）、`invalid_json`、`provider_http_error` 等 code；每阶段调用次数也记在 `usage_json`。前端错误区目前只渲染 `message`，用户看不到"失败在哪个阶段、重跑会重新计费哪几步"。

期望效果：重启后打开导入抽屉，卡死的 job 自动变为"已中断，可继续"；失败时显示阶段、错误类别与原始片段；点"继续识别"从断点续跑。**不做**分布式任务队列，不改本地线程模型（关于批量图片"部分页失败"：当前批量模式是整批一次 LLM 调用（`job_worker.py` 第 250-271 行），不存在按页粒度，断点粒度保持"阶段级"即可，不要引入按页重试）。

### 后端部分

#### 步骤 1：新增"僵尸 running 任务"回收函数

打开 `apps/api/src/memory_anki/modules/palaces/application/mindmap_import/job_repository.py`，在 `list_jobs`（第 48 行）上方新增：

```python
def reconcile_stale_running_jobs(
    session: Session,
    *,
    is_thread_alive_fn,
    entity_key: str | None = None,
) -> int:
    """把没有存活线程的 running 任务置为 interrupted（服务重启后遗留）。"""
    query = session.query(MindMapImportJob).filter(
        MindMapImportJob.status == "running",
        MindMapImportJob.deleted_at.is_(None),
    )
    if entity_key is not None:
        query = query.filter(MindMapImportJob.entity_key == entity_key)
    changed = 0
    for job in query.all():
        if is_thread_alive_fn(job.id):
            continue
        progress = json_load(job.progress_json, empty_progress())
        progress["message"] = "识别被中断（服务重启），可继续识别，已完成的步骤不会重复调用 AI。"
        job.status = JOB_STATUS_INTERRUPTED
        job.pause_requested = False
        job.progress_json = json_dump(progress)
        job.updated_at = utc_now_naive()
        changed += 1
    if changed:
        session.commit()
    return changed
```

（`JOB_STATUS_INTERRUPTED` 已在本文件第 26 行 import；`json_load/json_dump/empty_progress/utc_now_naive` 均已 import。）

自查点：`cd apps/api && python -m pytest tests/test_mindmap_import_job_service.py -x` 通过（确认没改坏既有函数）。

#### 步骤 2：在服务门面接入回收（读取 job 前先回收）

打开 `apps/api/src/memory_anki/modules/palaces/application/mindmap_import_job_service.py`。该文件已经持有线程注册表 `_RUNNING_JOB_THREADS` 与 `_RUNNING_JOB_LOCK`（第 17-18 行）。修改 `get_job` 与 `list_jobs` 门面（第 112-119 行）：

```python
# 新增（放在 get_job 上方）
def _is_job_thread_alive(job_id: str) -> bool:
    with _RUNNING_JOB_LOCK:
        thread = _RUNNING_JOB_THREADS.get(job_id)
    return bool(thread and thread.is_alive())


def reconcile_stale_running_jobs(session, *, entity_key: str | None = None) -> int:
    from .mindmap_import.job_repository import reconcile_stale_running_jobs as _reconcile

    return _reconcile(
        session,
        is_thread_alive_fn=_is_job_thread_alive,
        entity_key=entity_key,
    )


# 修改前
def get_job(*args, **kwargs):
    _sync_facade_dependencies()
    return _job_api.get_job(*args, **kwargs)

# 修改后
def get_job(*args, **kwargs):
    _sync_facade_dependencies()
    session = args[0] if args else kwargs.get("session")
    if session is not None:
        reconcile_stale_running_jobs(session)
    return _job_api.get_job(*args, **kwargs)
```

**行为验收（操作 → 期望现象）：**

1. 创建单图导入任务并运行，在"结构识别"阶段进行中强杀后端进程 → 重启后端，打开导入抽屉 → 该任务显示"已中断"，带"继续识别"按钮，不再无限转圈。
2. 点"继续识别" → 任务从中断阶段继续；若结构树已落盘（`structure_tree.json` 存在），不再产生新的结构识别 AI 调用（`usage.structure` 不增加）。
3. 断网后运行任务 → 任务失败，错误明细块显示"阶段：结构识别"、`provider_network_error` 对应的中文 message、原始片段与"已完成 N 次 AI 调用"提示。
