# 05-14 客户端偏好后端权威化收尾

## 1. 原始需求

客户端偏好迁移到后端后，旧 `localStorage` 中的 `break_guard_config` 仍可能存在。目标是在启动 bootstrap 中迁移旧值，同时避免用默认值覆盖服务端权威配置。

## 2. 详细执行清单

1. 修改 `apps/web/src/app/providers/useClientPreferenceBootstrap.ts`，提取 `bootstrapClientPreferences()`。
2. 启动时读取后端偏好，若服务端无值再迁移 legacy `break_guard_config`。
3. 使用统一 app event 通知偏好更新。
4. 补 `apps/web/src/app/providers/useClientPreferenceBootstrap.test.ts`。

## 3. 测试验收标准

- `cd apps/web && npm run test -- src/app/providers/useClientPreferenceBootstrap.test.ts` 通过。
- `cd apps/web && npm run typecheck` 通过。
