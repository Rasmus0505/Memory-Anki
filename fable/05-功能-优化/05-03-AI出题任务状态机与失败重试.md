# 05-03 AI 出题任务状态机与失败重试

## 1. 原始需求

AI 出题（图片/文本/复习脑图）目前是"一锤子买卖"，中断即全部作废，且进度提示是死代码：

1. **生成是单个同步 POST，无任务化**。四个生成端点（`apps/api/src/memory_anki/modules/palace_quiz/presentation/router.py` 第 291-395 行：`/palaces/{id}/quiz-generation/images`、`/text-files`、`/review-mindmap`、`/chapters/{id}/quiz-generation/outline`）都在一个 HTTP 请求内完成"准备 prompt → 阻塞调用 LLM → 解析入预览"。LLM 慢时请求可能挂几分钟，期间关掉页面/断网/前端超时，预览就丢了。
2. **LLM 结果其实已经落库，但没有恢复入口**。每次调用都通过 `begin/complete_external_ai_call_log` 写入 `external_ai_call_logs`（`modules/palace_quiz/application/ai_service_runtime_logging.py` 第 23-43 行，含完整 `response_text`），预览里也带 `ai_call_log_id`。也就是说：**请求中断后钱已经花了、答案已经存了，用户却只能重新上传重新付费**。仓库里甚至已有"从日志重建结果"的先例：`quiz_generation_child_chapter_log_reuse.py`（第 60-97 行）会扫描成功日志、复用分组结果。
3. **前端"流式进度"是断线的**。`apps/web/src/features/palace-quiz/quizGenerationController.ts` 的 `QuizGenerationRequestConfig` 声明了 `onStatus`/`onDelta` 回调（第 44-45 行），`usePalaceQuizGeneration.ts` 也维护了 `generationStreamStatus/StepLabel/PreviewText` 三个状态（第 75-77 行）并传入回调（第 213-221 行），但 `generatePalaceQuizPreview`（第 119-154 行）**从未调用这两个回调**——生成期间 UI 永远停在初始文案。
4. **失败状态没有分类**。`usePalaceQuizGeneration.ts` 第 240-245 行把所有异常揉成一个 `generationError` 字符串，用户无法区分"可直接重试"（网络断）、"可从日志恢复"（LLM 已成功但解析/传输失败）和"需要改配置"（模型未配置）。

期望效果：给生成流程一个明确的状态机（在文档中定义、在代码中落实），失败时前端能：a) 一键用原配置重试（文件还在内存里）；b) 检查最近的成功 AI 日志并**免费**恢复预览。**不做**后台任务队列/WebSocket 推送——单用户本地场景，同步请求 + 日志恢复已够用。

### 目标状态机（唯一权威定义）

| 状态 | 含义 | 进入条件 | 可转移到 |
|---|---|---|---|
| `idle` | 未开始 | 初始/清空预览 | `generating` |
| `generating` | 请求进行中 | 点击生成 | `preview_ready`、`failed` |
| `preview_ready` | 预览已就绪 | 请求成功 | `saving`、`idle`（丢弃） |
| `saving` | 写入题库中 | 点击保存 | `saved`、`preview_ready`（保存失败可重试） |
| `saved` | 已入库 | 保存成功 | `idle` |
| `failed` | 生成失败 | 请求异常 | `generating`（重试）、`preview_ready`（从日志恢复成功） |

```python
"""Recover a quiz generation preview from a stored external AI call log."""

from __future__ import annotations

from typing import Any

from sqlalchemy.orm import Session

from memory_anki.infrastructure.llm.external_ai_call_logs import get_external_ai_call_log

from ._question_utils import normalize_generated_question_drafts
from .quiz_generation_errors import PalaceQuizGenerationError
from .quiz_generation_preview_result import build_quiz_generation_preview_result

RECOVERABLE_OPERATIONS = {
    "palace_quiz_generate_images",
    "palace_quiz_generate_text_files",
    "palace_quiz_generate_review_mindmap",
    "chapter_quiz_generate_outline",
}


def recover_quiz_preview_from_log(
    session: Session,
    *,
    palace_id: int,
    log_id: str,
) -> dict[str, Any]:
    payload = get_external_ai_call_log(session, log_id)
    if not payload:
        raise PalaceQuizGenerationError("AI 调用日志不存在。")
    if payload.get("status") != "success":
        raise PalaceQuizGenerationError("该日志不是成功记录，无法恢复预览。")
    if payload.get("operation") not in RECOVERABLE_OPERATIONS:
        raise PalaceQuizGenerationError("该日志不是出题生成记录。")
    if payload.get("palace_id") not in (None, palace_id):
        raise PalaceQuizGenerationError("日志与当前宫殿不匹配。")

    response_text = str((payload.get("response_payload") or {}).get("response_text") or "")
    if not response_text.strip():
        raise PalaceQuizGenerationError("日志中没有可用的模型返回文本。")

    source_meta = {"ai_call_log_id": log_id, "recovered_from_log": True}
    drafts, warnings, generation_stats = normalize_generated_question_drafts(
        response_text,
        source_meta=source_meta,
    )
    return build_quiz_generation_preview_result(
        scope_key="palace_id",
        scope_id=palace_id,
        questions=drafts,
        source_meta=source_meta,
        log_id=log_id,
        warnings=[*warnings, "本预览由历史 AI 日志恢复，OCR 溯源与关卡分组信息不含在内。"],
        generation_stats=generation_stats,
        grouped_questions=None,
        resolved_ai=None,
        extra_fields={"ocr_sources": [], "recovered_from_log": True},
    )


__all__ = ["recover_quiz_preview_from_log"]
```

const handleRetryLastGeneration = async () => {
  if (!lastFailedConfig) return
  await executeGenerationPreview(lastFailedConfig)
}

const handleRecoverFromLog = async () => {
  if (!palaceId) return
  setGenerationLoading(true)
  try {
    const { items } = await listAiCallLogsApi({ palaceId, status: 'success', limit: 5 })
    const candidate = items.find((item) => item.operation.startsWith('palace_quiz_generate'))
    if (!candidate) {
      toast.message('最近没有可恢复的出题日志。')
      return
    }
    const preview = await recoverPalaceQuizPreviewFromLogApi(palaceId, candidate.id)
    setGenerationPreview(preview)
    setGenerationError('')
    toast.success('已从 AI 调用日志恢复预览（未产生新的 AI 费用）。')
  } catch (error) {
    toast.error(error instanceof Error ? error.message : '恢复失败。')
  } finally {
    setGenerationLoading(false)
  }
}

**行为验收（操作 → 期望现象）：**

1. 正常生成一次图片出题并保存 → 流程与改动前一致，历史记录出现新项。
2. 生成过程中（LLM 响应前）强制刷新页面 → 等约 1 分钟后打开该宫殿做题页，点历史项旁"恢复该次预览"或错误区"从 AI 日志恢复预览" → 预览出现，带"由历史 AI 日志恢复"警告，不产生新的 AI 调用（`GET /ai-call-logs` 无新增记录）。
3. 断网状态点生成 → 进入失败态，出现"重试本次生成"按钮；恢复网络后点击 → 用同一批文件与配置重新发起请求（无需重新选文件）。
