# 05-02 复习阶段进度修复自动化

## 1. 原始需求

本文件是 `05-02-复习阶段进度修复流程自动化.md` 的重复文档，已同步为同一套**计数版**实现方案：只新增 `GET /review/stage-progress-health`，不新增 `/review/stage-progress/health` 双轨接口。

复习阶段进度数据可能因中断、跨设备同步或历史 bug 变脏（孤儿进度、孤儿学习会话、阶段序列断档）。此前修复入口只在设置页，普通复习路径看不到"需要修复"信号；一次性启动迁移也只会跑一次，之后新产生的脏数据不会被发现。

期望效果：后端提供一个**只读**健康检查接口；服务启动时在既有 warmup 线程里自检并记日志；前端复习总览页发现异常时显示横幅，用户一键触发既有 `POST /review/repair-stage-progress`。不做启动自动修复，设置页原按钮保留。

## 2. 详细执行清单

> 硬性约束：**不要**修改 `repair_review_stage_progress` 本体及其四个 `_migrate/_recover/_sync` 私有函数；**不要**改动一次性迁移和迁移标记；**不要**让启动流程自动执行重建（只做只读检测）；**不要**删除设置页的手动修复按钮；不新增数据库表；不新增 `/review/stage-progress/health`。

### 后端部分

#### 步骤 1：新增只读计数检测函数

在 `apps/api/src/memory_anki/modules/reviews/application/review_execution_service.py` 的 `repair_review_stage_progress` 上方新增 `detect_review_stage_progress_issues(session: Session) -> dict`。函数只做 SELECT/内存统计，不调用任何修复函数，返回：

```python
{
    "orphan_progress_count": orphan_progress_count,
    "orphan_study_session_count": orphan_study_session_count,
    "stage_gap_palace_count": stage_gap_palace_count,
    "total_issues": total_issues,
    "needs_repair": total_issues > 0,
}
```

计数规则与主文档一致：未完成 review `SessionProgress` 指向空/不存在 schedule 计为孤儿进度；active review `StudySession` 指向空/不存在 schedule 计为孤儿会话；同一宫殿存在"未完成阶段编号 < 已完成阶段编号"计为阶段断档。

#### 步骤 2：新增健康检测路由

打开 `apps/api/src/memory_anki/modules/reviews/presentation/router.py`，在 `api_repair_review_stage_progress` 上方新增，并把 `detect_review_stage_progress_issues` 加进既有 import 块：

```python
@router.get("/review/stage-progress-health")
def api_review_stage_progress_health(session: Session = Depends(session_dep)):
    return {"ok": True, **detect_review_stage_progress_issues(session)}
```

**不要**改动既有 `POST /review/repair-stage-progress` 的行为与响应结构（`ProfileSettingsPage.tsx` 第 117-124 行依赖 `palace_count` / `practice_recovery_count` / `orphan_progress_count` / `orphan_study_session_count` / `study_session_count` 字段）。

自查点：启动后端后 `curl http://127.0.0.1:8012/api/v1/review/stage-progress-health` 返回 JSON，包含 `needs_repair` 与三个计数字段。

#### 步骤 3：startup warmup 只读自检并写日志

打开 `apps/api/src/memory_anki/app/startup_warmup.py`，在 `run_startup_warmup` 的 `logger.info("startup warmup completed")` 之前插入：
