# 05-02 复习阶段进度修复自动化

## 1. 原始需求

复习阶段进度修复目前有两个入口，均已核实：

1. **一次性启动迁移**：`apps/api/src/memory_anki/app/startup_runtime.py` 的 `run_review_schedule_repair_migration`（第 63-73 行）在启动时调用 `repair_review_stage_progress`，但被迁移键 `review_schedule_anchor_repair_v1`（第 46 行）保护，**只会执行一次**，之后再出现数据不一致（如手工改库、异常中断导致 schedule 缺失）不会被发现。
2. **手动按钮**：前端只有设置页 `apps/web/src/features/profile/ProfileSettingsPage.tsx` 的 `handleRepairReviewStageProgress`（第 112-142 行）调 `POST /review/repair-stage-progress`，用户需要先知道"数据坏了"才会去点，而问题实际暴露在复习页（队列里少宫殿、进度卡住）。

修复本体 `repair_review_stage_progress`（`modules/reviews/application/review_execution_service.py` 第 149-162 行）会调用 `rebuild_all_pending_review_schedules`（`schedule_rebuild_service.py` 第 272-312 行，全文件 509 行）**重建所有宫殿的全部 schedule**，成本较高且有覆盖风险，不适合每次启动无脑执行。

期望效果：新增一个**只读的健康检测**，启动时自动执行并记录；复习总览页发现异常时展示提示条并提供一键修复入口；设置页原按钮保留。

#### 步骤 1：新增只读健康检测函数

打开 `apps/api/src/memory_anki/modules/reviews/application/review_execution_service.py`，在 `repair_review_stage_progress`（第 149 行）上方新增：

```python
def detect_review_stage_health(session: Session) -> dict:
    """只读检测复习阶段进度是否损坏，不修改任何数据。"""
    issues: list[dict] = []
    palaces = session.query(Palace).all()
    for palace in palaces:
        schedules = sorted(
            list(palace.review_schedules or []),
            key=lambda item: (item.review_number, item.id),
        )
        algorithm = normalize_algorithm(
            next((s.algorithm_used for s in schedules if s.algorithm_used), "ebbinghaus")
        )
        total = len(get_algorithm_intervals(session, algorithm))
        completed_numbers = sorted(
            {int(s.review_number) for s in schedules if s.completed}
        )
        pending = [s for s in schedules if not s.completed]

        reason = None
        if not palace.mastered and schedules and not pending:
            reason = "未掌握但没有任何待复习排程"
        elif completed_numbers and completed_numbers != list(range(len(completed_numbers))):
            reason = "已完成阶段序号存在空洞"
        elif len({int(s.review_number) for s in pending}) != len(pending):
            reason = "存在重复的待复习阶段"
        elif palace.mastered and len(completed_numbers) < total and schedules:
            reason = "已标记掌握但完成阶段数不足"
        if reason:
            issues.append({"palace_id": palace.id, "title": palace.title, "reason": reason})
    return {"healthy": not issues, "issue_count": len(issues), "issues": issues}
```

依赖的 `Palace`、`normalize_algorithm`、`get_algorithm_intervals` 已在本文件顶部（第 12-30 行）导入，无需新增 import。

自查点：`cd apps/api && python -c "from memory_anki.modules.reviews.application.review_execution_service import detect_review_stage_health"` 无 ImportError；对干净测试库调用返回 `{"healthy": True, "issue_count": 0, "issues": []}`。

**行为验收（操作 → 期望现象）：**

1. 干净数据库启动 → `GET /review/stage-progress/health` 返回 `healthy: true`，复习总览页无警告卡片。
2. 手工制造损坏（如 SQL：把某未掌握宫殿的所有 `review_schedules.completed` 置 1 且 `palaces.mastered` 保持 0）→ 刷新复习总览页，出现"检测到 1 个宫殿的复习进度异常"卡片。
3. 点击"一键修复" → toast 显示"修复完成：重建 N 个宫殿"，卡片消失，该宫殿重新出现在正确的复习阶段。
4. 重启后端 → 启动日志（logger `memory_anki.startup`）在数据损坏时输出 warning，正常时无该 warning。
5. 设置页"复习配置"Tab 的原修复按钮 → 点击后仍按原文案（"重建 N 个宫殿…"）提示成功。
