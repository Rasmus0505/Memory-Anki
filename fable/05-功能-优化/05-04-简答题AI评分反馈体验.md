# 05-04 简答题 AI 评分反馈体验

## 1. 原始需求

简答题的 AI 点评目前是一整段自由文本，无法快速看出"哪些要点答到了、哪些漏了"：

1. **Prompt 明确禁止结构化输出**。默认提示词 `PALACE_QUIZ_SHORT_ANSWER_FEEDBACK_PROMPT`（`apps/api/src/memory_anki/modules/settings/application/ai_prompt_templates/__init__.py` 第 130-137 行）写的是"直接输出点评正文，不要输出 JSON"。模型输入只有四个字段：题干、学生答案、参考答案、解析（`modules/palace_quiz/application/quiz_generation_feedback_request_payload.py` 第 17-25 行）。
2. **返回结构只有一个文本字段**。`generate_short_answer_feedback`（`quiz_generation_feedback.py` 第 20-48 行）以 `response_format=None` 调用 LLM，返回 `{"question_id", "feedback_text", "ai_call_log_id", "resolved_ai"}`；前端契约 `PalaceShortAnswerFeedback`（`apps/web/src/shared/api/contracts/quiz.ts` 第 183-188 行）同样只有 `feedback_text`。
3. **前端整段展示**。`apps/web/src/features/palace-quiz/QuizQuestionInteraction.tsx` 第 593-610 行把 `feedback_text` 用一个 `whitespace-pre-wrap` div 全量渲染，读长点评时找不到重点。该组件同时被 freestyle 信息流复用（`features/freestyle/FreestylePage.tsx` 第 1220-1251 行走同一个 API 与组件）。

期望效果：点评返回结构化维度——结论（正确/部分正确/错误）、命中要点列表、遗漏要点列表、改进建议——前端按区块着色展示；同时**必须向后兼容**：模型不按 JSON 输出（例如用户自定义了提示词）时回退为整段文本展示，行为与现在完全一致。

## 2. 详细执行清单

> 硬性约束：**不要**改 `quiz_generation_feedback_request.py` 的场景 key（`quiz_short_answer_feedback`）、温度与超时；**不要**强制 `response_format={"type": "json_object"}`（用户可能自定义了非 JSON 提示词，强制 JSON 模式会让请求直接报错）；**不要**动 `explain_question` 讲解流程。

并在文件内（`generate_short_answer_feedback` 上方）新增解析函数：

```python
import json
import re


def _parse_structured_feedback(response_text: str) -> dict | None:
    """尝试从模型输出提取结构化点评；失败返回 None（回退纯文本）。"""
    text = response_text.strip()
    match = re.search(r"\{[\s\S]*\}", text)
    if not match:
        return None
    try:
        payload = json.loads(match.group(0))
    except Exception:
        return None
    if not isinstance(payload, dict):
        return None
    verdict = str(payload.get("verdict") or "").strip()
    if verdict not in {"correct", "partial", "incorrect"}:
        return None
    hit_points = [str(item) for item in payload.get("hit_points") or [] if str(item).strip()]
    missed_points = [str(item) for item in payload.get("missed_points") or [] if str(item).strip()]
    suggestion = str(payload.get("suggestion") or "").strip()
    fallback_lines = []
    if hit_points:
        fallback_lines.append("答到的要点：\n" + "\n".join(f"- {item}" for item in hit_points))
    if missed_points:
        fallback_lines.append("遗漏或有偏差：\n" + "\n".join(f"- {item}" for item in missed_points))
    if suggestion:
        fallback_lines.append(f"建议：{suggestion}")
    return {
        "verdict": verdict,
        "hit_points": hit_points,
        "missed_points": missed_points,
        "suggestion": suggestion,
        "fallback_text": "\n\n".join(fallback_lines) or text,
    }
```

`feedback_text` 始终有值（结构化成功时是拼接文本），保证旧前端/离线缓存不炸。**不要**修改 `_call_logged_chat_completion` 的调用参数。

自查点：Python REPL 里分别用 JSON 字符串、纯文本、`verdict` 非法三种输入调 `_parse_structured_feedback`，得到 dict、None、None。
