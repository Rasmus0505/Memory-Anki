# 05-06 知识章节与宫殿绑定联动优化

## 1. 原始需求

章节-宫殿绑定的数据流已核实：`PUT /palaces/{id}/chapters`（`apps/api/src/memory_anki/modules/knowledge/presentation/router.py` 第 286-299 行）调用 `set_palace_chapter_links`（`modules/palaces/application/palace_chapter_binding.py` 第 29-54 行，显式勾选自动扩展到全部祖先章节）+ `reconcile_palace_chapter_binding`（同文件第 125 行起，维护 `primary_chapter_id`）。当前有三个体验缺口：

1. **章节页看宫殿"看得到但信息太少"**。`GET /chapters/{id}`（router.py 第 169-201 行）的 `palace_out`（第 176-180 行）只返回 `id/title/pegs`，前端 `apps/web/src/features/knowledge/KnowledgePage.tsx` 第 468-474 行只能渲染纯标题链接——看不出哪个宫殿已掌握、哪个复习到第几阶段。反向（宫殿页看章节）已经很完善（`features/palace-edit/components/PalaceChapterPanel.tsx`，269 行）。
2. **解绑零保护**。在 `PalaceChapterPanel` 取消勾选章节会立即调 `PUT /palaces/{id}/chapters` 覆盖全部绑定（面板第 116-122 行 checkbox 直接 `onToggleChapter`），没有任何确认；解绑后该宫殿会从章节复习队列（`/review/chapter/{id}/queue`）和章节做题范围里消失，用户无感知。
3. **删除章节是静默级联毁灭**。`DELETE /chapters/{id}`（router.py 第 250-265 行）递归删除整棵子树；而做题题目的 `source_chapter_id` 外键是 `ondelete="CASCADE"`（`infrastructure/db/_tables/palaces.py` 第 449-452 行）——删除章节会**连带删掉挂在子树上的所有题目**和 `chapter_palaces` 绑定行，前端没有展示影响范围的确认。

期望效果：章节页宫殿列表带掌握/复习进度徽章；取消绑定时提示影响；删除章节前返回"子树内绑定宫殿数 + 题目数"，前端二次确认。

## 2. 详细执行清单

> 硬性约束：**不要**改 `set_palace_chapter_links` 的祖先扩展语义与 `reconcile_palace_chapter_binding` 的主章节推导（`test_palace_chapter_binding.py` 有完整用例护住）；**不要**改绑定表结构；**不要**把删除改成软删除（超出范围）。

#### 步骤 1：章节详情的宫殿列表补充复习状态

打开 `apps/api/src/memory_anki/modules/knowledge/presentation/router.py`，修改 `get_chapter` 内的 `palace_out`（第 176-180 行）：

```python
# 修改前
    def palace_out(p):
        return {
            "id": p.id, "title": p.title,
            "pegs": [{"id": pg.id, "name": pg.name, "content": pg.content} for pg in p.pegs],
        }

# 修改后
    def palace_out(p):
        schedules = list(p.review_schedules or [])
        completed = sum(1 for item in schedules if item.completed)
        pending_dates = sorted(
            item.scheduled_date for item in schedules if not item.completed
        )
        return {
            "id": p.id, "title": p.title,
            "pegs": [{"id": pg.id, "name": pg.name, "content": pg.content} for pg in p.pegs],
            "mastered": bool(p.mastered),
            "archived": bool(p.archived),
            "review_stage_completed": completed,
            "review_stage_total": len(schedules),
            "next_due_date": pending_dates[0].isoformat() if pending_dates else None,
        }
```

新字段全部只增不改，旧前端不受影响。**不要**在这里调 `palace_detail_json`（会拖入 editor_doc 大字段，章节页不需要）。

#### 步骤 2：删除章节改为"先报影响，force 才删"

同文件，替换 `delete_chapter`（第 250-265 行）：

```python
@router.delete("/chapters/{chapter_id}")
def delete_chapter(chapter_id: int, force: bool = False, s: Session = Depends(session_dep)):
    c = s.query(Chapter).filter_by(id=chapter_id).first()
    if not c:
        return {"ok": True}

    def _collect_subtree_ids(chapter: Chapter) -> list[int]:
        ids = [chapter.id]
        for child in chapter.children or []:
            ids.extend(_collect_subtree_ids(child))
        return ids

    subtree_ids = _collect_subtree_ids(c)
    linked_palace_count = (
        s.execute(
            text("SELECT COUNT(DISTINCT palace_id) FROM chapter_palaces WHERE chapter_id IN :ids"),
            {"ids": tuple(subtree_ids)},
        ).scalar() or 0
    )
    question_count = (
        s.query(PalaceQuizQuestion)
        .filter(PalaceQuizQuestion.source_chapter_id.in_(subtree_ids))
        .count()
    )
    if not force and (linked_palace_count > 0 or question_count > 0):
        return JSONResponse(
            status_code=409,
            content={
                "ok": False,
                "requires_force": True,
                "chapter_count": len(subtree_ids),
                "linked_palace_count": int(linked_palace_count),
                "question_count": int(question_count),
            },
        )
    _delete_recursive(c, s)
    s.commit()
    maybe_create_rolling_backup("rolling-delete-chapter")
    return {"ok": True}
```

需要在文件顶部补 import：`from sqlalchemy import text` 与 `from memory_anki.infrastructure.db.models import PalaceQuizQuestion`（先确认该模型确实从 `models` 导出，`rg "PalaceQuizQuestion" apps/api/src/memory_anki/infrastructure/db/models*` 核实）。注意 SQLite 的 `IN :ids` 元组绑定：若报错，改用 `text(...).bindparams(bindparam("ids", expanding=True))` 写法（`from sqlalchemy import bindparam`）。保留 `_delete_recursive` 不动。

自查点：对一个绑定了宫殿的章节 `curl -X DELETE .../chapters/{id}` 返回 409 与计数；追加 `?force=true` 后删除成功。

#### 步骤 6：删除章节的影响确认

打开 `apps/web/src/features/knowledge/KnowledgePage.tsx`，找到调用删除章节 API 的处理函数（搜索 `deleteChapter`）。改为两段式：

```bash
# 期望：409 + {"requires_force": true, "linked_palace_count": N, "question_count": M}
```

**行为验收（操作 → 期望现象）：**

1. 知识大纲页选中一个绑定了宫殿的章节 → 右侧宫殿列表每项显示"已掌握"或"复习 x/y"徽章与下次复习日期。
2. 宫殿编辑页取消勾选一个已关联章节 → 弹出确认文案；点取消则勾选状态不变，点确定则解绑并刷新。
3. 删除一个下挂宫殿和题目的章节 → 弹出"共 N 个章节、关联 M 个宫殿、K 道题目"确认；确认后删除成功，取消则无变化。
4. 删除一个空章节 → 直接删除，无确认弹窗（无影响则不打扰）。

**回归检查：**

- 绑定/解绑后 `primary_chapter_id` 推导逻辑不变（`test_palace_chapter_binding.py` 既有用例全过）。
- 章节复习队列 `/review/chapter/{id}/queue`、章节出题 `/chapters/{id}/quiz-generation/outline` 不受影响。
