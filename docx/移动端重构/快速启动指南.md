## 如何访问移动端

### 方式1：直接访问路径（开发/测试）
```
http://localhost:5173/m/
http://localhost:5173/m/freestyle
http://localhost:5173/m/palaces
http://localhost:5173/m/review
http://localhost:5173/m/english
http://localhost:5173/m/profile
```

### 方式2：添加到主屏幕（生产环境）

#### iOS (Safari)
1. 在 Safari 中打开应用网址
2. 点击底部分享按钮 📤
3. 选择"添加到主屏幕"
4. 点击"添加"
5. 从主屏幕启动 → 自动进入移动端 ✅

#### Android (Chrome)
1. 在 Chrome 中打开应用网址
2. 点击右上角菜单 ⋮
3. 选择"添加到主屏幕"或"安装应用"
4. 点击"添加"
5. 从主屏幕启动 → 自动进入移动端 ✅

## 常见问题

### Q: 为什么从主屏幕启动没有进入移动端？
A: 确保：
1. 是从主屏幕图标启动（不是浏览器书签）
2. Service Worker 已注册（生产环境）
3. Manifest 正确配置

### Q: 底部 Tab Bar 没有显示？
A: 检查：
1. 是否访问了 `/m/*` 路径
2. 是否在全屏模式页面（练习、复习会话）
3. 控制台是否有错误

### Q: Safe area (刘海区域) 显示不正确？
A: 确保：
1. 以 PWA standalone 模式运行
2. `viewport-fit=cover` 已设置（index.html）
3. CSS 变量 `--mobile-safe-area-inset-*` 生效

### Q: 点击按钮没有反馈？
A: 检查：
1. 按钮是否有 `mobile-touch-button` 类
2. 是否满足 44px 最小尺寸
3. CSS `:active` 伪类是否生效

### Q: 如何切换回桌面端？
A: 
- 方式1：访问非 `/m/*` 路径（如 `/dashboard`）
- 方式2：在浏览器中打开（非 standalone 模式）

## 性能优化提示

### 首屏加载优化
- 页面懒加载已启用（Suspense + lazy）
- 图片使用懒加载（loading="lazy"）
- 避免大文件打包

### 滚动性能
- 使用 `-webkit-overflow-scrolling: touch`
- 避免复杂的 CSS 动画
- 使用 `will-change` 提示浏览器

### 内存优化
- 及时清理事件监听器
- 使用虚拟滚动处理长列表
- 避免内存泄漏
