# 移动端 PWA 重构总体方案

## 目标
彻底重做移动端 PWA 体验，解决当前存在的问题：
- 显示和交互不舒服
- 功能不完整（只有随心模式）
- 页面跳转错误

## 用户选择的方案

### 功能范围
- ✅ 随心模式（Freestyle）
- ✅ 宫殿练习（Palace Quiz/Practice）
- ✅ 复习系统（Review）
- ✅ 英语学习（English）

### 导航方案
**底部 Tab Bar**（iOS/Android 原生风格）
```
┌──────────────────────────────┐
│                              │
│         页面内容区域          │
│                              │
├──────────────────────────────┤
│  🎲随心  🏛宫殿  📖复习  🌐英语  👤我的 │
└──────────────────────────────┘
```

### 架构方案
**专属移动端页面**：为每个功能模块写独立的移动端版本，与桌面端解耦

### 启动入口
**新增 standalone 模式自动检测** + 保留 `/m/*` 路径入口

## 新的文件结构

```
apps/web/src/features/mobile/
  MobileApp.tsx                     ← 移动端根组件（替换 MobileFreestyleApp.tsx）
  MobileAppRouter.tsx               ← 移动端路由配置
  
  components/
    layout/
      MobileLayout.tsx              ← 移动端通用布局（处理 safe area）
    nav/
      MobileTabBar.tsx              ← 底部 Tab Bar 导航
      MobileTopBar.tsx              ← 顶部标题栏（可选）
  
  pages/
    freestyle/
      MobileFreestylePage.tsx       ← 随心模式移动版
    palace/
      MobilePalaceListPage.tsx      ← 宫殿列表
      MobilePalaceDetailPage.tsx    ← 宫殿详情/练习/测验
    review/
      MobileReviewPage.tsx          ← 复习系统移动版
    english/
      MobileEnglishPage.tsx         ← 英语学习移动版
    profile/
      MobileProfilePage.tsx         ← 个人/设置页
```

## 路由结构

```
/m                    → 重定向到 /m/freestyle
/m/freestyle          → MobileFreestylePage
/m/palaces            → MobilePalaceListPage
/m/palaces/:id        → MobilePalaceDetailPage
/m/review             → MobileReviewPage
/m/english            → MobileEnglishPage
/m/profile            → MobileProfilePage
```

## 实施步骤（按顺序执行）

1. **01-添加移动端CSS样式.md** - 在 index.css 中添加移动端专属样式

3. **03-创建底部导航栏.md** - MobileTabBar.tsx（5个 Tab）
4. **04-创建随心模式页面.md** - MobileFreestylePage.tsx
5. **05-创建宫殿列表页面.md** - MobilePalaceListPage.tsx
6. **06-创建宫殿详情页面.md** - MobilePalaceDetailPage.tsx
7. **07-创建复习页面.md** - MobileReviewPage.tsx
8. **08-创建英语学习页面.md** - MobileEnglishPage.tsx

## 设计原则

### 触控友好
- 最小点击区域：44px × 44px
- 按钮间距：至少 8px
- 避免依赖 hover 状态

### iOS 适配
- 支持 safe area insets（刘海、Home 指示条）
- 支持触感反馈（Haptic Feedback）
- 尊重系统主题色

### 性能优化
- 懒加载页面组件
- 虚拟滚动长列表
- 图片懒加载

### 离线支持
- Service Worker 缓存策略（保持现有）
- 离线状态提示
- 本地数据优先

## 注意事项

1. **不改变启动方式**：保留现有的 `/m/*` 路径检测 + PWA manifest
2. **不碰桌面端**：所有修改限制在 `features/mobile/` 目录
3. **渐进增强**：先完成基础功能，后续可添加手势、动画
4. **测试覆盖**：每个页面需要基本的测试用例

## 技术栈

- React Router（路由）
- Tailwind CSS（样式）
- Lucide React（图标）
- 复用 shared 目录的工具函数和 hooks
