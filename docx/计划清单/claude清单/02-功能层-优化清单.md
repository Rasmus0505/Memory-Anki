# 功能层 - 优化清单

> **优化原则**：从工具到体验，从功能到价值

---

## ⚡ 核心功能优化方向

### 1. 记忆宫殿编辑器 - 从工具到体验

#### 当前问题

**技术层面**：
- 思维导图编辑器隔离在 iframe 中，带来巨大复杂度
- HMR 被禁用，开发体验差
- `editor_doc` / `editor_config` / `editor_local_config` 三层配置混乱
- 宿主-iframe 桥接（hostBridge）、同步控制器（syncController）、反馈协调器（feedbackAudioCoordinator）

**用户体验层面**：
- 需要学习"Peg/Segment/MiniPalace"等技术概念
- "编辑"和"练习"是分离的，无法无缝切换
- 节点操作反馈不明显（虽有音效但体验不够流畅）

#### 优化方向

##### 1.1 去概念化

**删除术语**：
| 技术术语 | 用户友好名称 | 说明 |
|---------|-------------|------|
| Peg | 知识点 | 宫殿中的每个节点 |
| Segment | 学习组 | 相关知识点的集合 |
| MiniPalace | 训练关卡 | 针对性的小型练习单元 |
| Palace | 记忆宫殿 | 保持，但加强空间隐喻 |

**UI 改造**：
```tsx
// 旧版：暴露技术细节
<button>创建分段</button>
<button>生成迷你宫殿</button>

// 新版：用户友好
<button>将选中内容组成学习组</button>
<button>创建专项训练</button>
```

##### 1.2 所见即所得 - 编辑即练习

**核心理念**：在编辑器中直接切换到"回忆模式"

**实现方案**：

**好处**：
- 编辑完立即可以自测效果
- 减少"编辑→保存→切换到练习页面"的流程摩擦
- 增强即时反馈感

```tsx

  <Button>手动调整</Button>
```

**智能拆分建议**：
```python
# AI 自动建议拆分粒度，不要让用户调参数
```

##### 1.4 去除 iframe 隔离（可选，高风险）

**当前方案**：
- 思维导图在 iframe 中，通过 `postMessage` 通信
- 复杂的桥接逻辑（hostBridge）

**理由**：
- 避免样式冲突
- 隔离全局状态

**问题**：
- 极大的复杂度
- HMR 无法工作
- 开发体验差

**优化方案**：

**方案A（推荐）：保留 iframe，但简化桥接**
```tsx
// 简化 hostBridge，只保留核心通信
const simplifiedBridge = {
  syncContent: (content) => iframe.postMessage({ type: 'sync', content }),
  onContentChange: (callback) => window.addEventListener('message', callback)
}
```

**方案B（激进）：去除 iframe，CSS Modules 隔离**
```tsx
// 用 CSS Modules 或 Shadow DOM 隔离样式
import styles from './MindMap.module.css'

<div className={styles.mindmapContainer}>
  <MindMapCanvas nodes={nodes} />
</div>
```

**建议**：先执行方案A，观察效果后再考虑方案B

### 2. 间隔重复系统 - 从算法到智能

#### 当前问题

**算法复杂但感知弱**：
- 有完整的艾宾浩斯实现（`ebbinghaus_strategy.py`）
- 但用户只是被动等待"到期复习"
- 缺乏主动式学习引导

**用户体验问题**：
- "逾期平滑"、"调度迁移"等底层细节暴露给用户
- 复习队列只是简单的列表，无优先级感知
- 无法预知"即将遗忘"的内容

#### 优化方向

##### 2.1 预测式复习

**核心理念**：根据遗忘曲线，提前推荐"即将遗忘"的内容

**实现方案**：
```python
class PredictiveReviewService:
    def get_daily_recommendations(self, user_id: int):
        """生成今日学习推荐"""
        
        # 1. 必须复习（已到期）
        due_now = self.get_due_items()
        
        # 2. 建议复习（即将遗忘，记忆强度 < 0.3）
        at_risk = self.get_items_at_risk()
        
        # 3. 可选复习（提前巩固，记忆强度 0.3-0.6）
        optional = self.get_optional_items()
        
        return {
            'must_review': due_now,        # 红色标记
            'should_review': at_risk,      # 黄色标记
            'can_review': optional,        # 绿色标记
            'estimated_time': self.estimate_time(due_now + at_risk)
        }
```

**前端展示**：
```tsx
<DailyRecommendation>
  <Section variant="urgent">
    <h3>🔴 必须复习（3个宫殿，约15分钟）</h3>
    <p>这些内容已经到期，建议立即复习</p>
  </Section>
  
  <Section variant="warning">
    <h3>🟡 建议复习（5个宫殿，约20分钟）</h3>
    <p>这些内容即将遗忘，现在复习效果最好</p>
  </Section>
  
  <Section variant="optional">
    <h3>🟢 可选复习（8个宫殿，约30分钟）</h3>
    <p>提前巩固，让记忆更牢固</p>
  </Section>
</DailyRecommendation>
```

##### 2.3 学习节奏调节

**智能负荷控制**：
```python
class AdaptiveScheduler:
    def adjust_daily_load(self, user_id: int):
        """根据用户状态调整复习量"""
        
        recent_performance = self.get_recent_performance(user_id)
        
        if recent_performance.accuracy < 0.6:
            # 正确率低 → 减少复习量，增加练习
            return {
                'review_count': 5,  # 减少
                'practice_mode': 'focus'  # 针对性练习
            }
        
        elif recent_performance.study_days_streak > 7:
            # 连续学习7天 → 适当增加挑战
            return {
                'review_count': 15,  # 增加
                'include_optional': True  # 包含可选复习
            }
        
        elif recent_performance.last_session_days_ago > 3:
            # 中断3天 → 降低压力，温和恢复
            return {
                'review_count': 3,  # 大幅减少
                'difficulty': 'easy'  # 只复习最基础的
            }
```

### 3. 题库系统 - 从刷题到巩固

#### 当前问题

**关联松散**：
- 题目可以不关联章节（`source_chapter_id` 可空）
- 题目与宫殿节点无精确对应
- 做错题后无法定位到具体知识点

**生成质量参差**：
- 多种生成方式（图片/PDF/复习导图/章节大纲）
- 无质量控制，生成即推送
- 缺乏"好题"的筛选机制

#### 优化方向

##### 3.1 强制关联 - 题目即标注

**数据模型改造**：
```python
class QuizQuestion:
```

##### 3.2 渐进式难度

**核心理念**：题目难度随记忆强度增长

**实现方案**：
```python
class ProgressiveDifficultyQuiz:
    def generate_question(self, node: Node, memory_strength: float):
        """根据记忆强度生成对应难度的题目"""
        
        if memory_strength < 0.3:
            # 初期：填空、选择
            return self.generate_fill_blank(node)
        
        elif memory_strength < 0.6:
            # 中期：简答、联想
            return self.generate_short_answer(node)
        
        elif memory_strength < 0.85:
            # 后期：案例应用
            return self.generate_case_study(node)
        
        else:
            # 精通：综合推理
            return self.generate_synthesis(node)
```

**题型模板**：

**初期 - 填空**：
```
题目：牛顿第一定律又称为______定律。
答案：惯性
```

**中期 - 简答**：
```
题目：请用自己的话解释"惯性"的含义。
评分：AI评分，检查关键词（物体、保持、运动状态）
```

**后期 - 案例**：
```
题目：一辆行驶的汽车突然刹车，车内乘客会向前倾，请用牛顿第一定律解释这一现象。
评分：AI评分，检查因果关系、概念应用
```

**精通 - 综合**：
```
题目：设计一个实验，验证"在没有外力作用时，物体保持静止或匀速直线运动"。
评分：AI评分，考察实验设计、变量控制
```

##### 3.3 AI 出题质量控制

**当前问题**：生成后直接推送，无质量检查

**优化方案**：

**生成后自检**：
```python
async def generate_quality_quiz(node: Node):
    """生成题目 + 质量检查"""
    
    # 1. 生成题目
    question = await llm.generate_question(node)
    
    # 2. 自检
    quality_check = await llm.check_quality(question, node)
    
    if quality_check.score < 0.7:
        # 质量不合格 → 重新生成
        question = await llm.generate_question(node, 
            feedback=quality_check.issues)
    
    # 3. 返回高质量题目
    return question

返回JSON：
{
    "score": 0.0-1.0,
    "issues": ["问题1", "问题2"],
    "suggestions": ["改进建议1"]
}
"""
```

### 4. 思维导图导入 - 从工具到工作流

#### 当前问题

**功能分散**：
- 图片识别（单张/批量）
- PDF导入（多种模式）
- 文本导入
- 每种都有独立的API和状态管理

**导入后负担重**：
- 用户需要手动调整
- 导入历史（mindmap_import_jobs）维护复杂
- 缺乏智能优化

#### 优化方向

##### 4.1 统一导入流

**核心理念**：无论什么格式，都走同一个智能处理管道

**实现方案**：
```python
class UnifiedImportService:
    async def import_content(self, content: Any):
        """统一导入接口"""
        
        # 1. 识别类型
        content_type = self.detect_type(content)
        
        # 2. 提取文本
        text = await self.extract_text(content, content_type)
        
        # 3. 智能处理
        palace = await self.process_intelligently(text)
        
        # 4. 返回预览
        return {
            'palace': palace,
            'preview': self.generate_preview(palace),
            'suggestions': self.generate_suggestions(palace)
        }
    
    async def process_intelligently(self, text: str):
        """智能识别学习意图"""
        
        intent = await self.detect_learning_intent(text)
        
        if intent == 'textbook_toc':
            # 教材目录 → 生成章节宫殿
            return self.generate_chapter_palace(text)
        
        elif intent == 'flowchart':
            # 流程图 → 生成步骤记忆
            return self.generate_process_palace(text)
        
        elif intent == 'concept_map':
            # 概念图 → 生成关联网络
            return self.generate_concept_palace(text)
        
        else:
            # 通用处理
            return self.generate_generic_palace(text)
```

```python
def generate_with_confidence(text: str):
```

##### 4.3 去除导入任务管理

**当前问题**：
- `MindMapImportJob` 表，维护状态机（pending/running/completed）
- 用户需要管理导入历史
- 增加复杂度

**优化方案**：
- 导入即完成，不保存"任务"
- 只保存"导入结果"（即生成的宫殿）
- 删除任务状态管理
