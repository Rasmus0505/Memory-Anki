# 功能层 - 删除清单

> **删除原则**：功能删除不是因为"做得不好"，而是"不应该做"

---

## 🗑️ 应该删除的功能模块

### 1. 英语听力模块（english）

**涉及文件**：
```
apps/api/src/memory_anki/modules/english/
apps/web/src/features/english/
apps/api/src/memory_anki/infrastructure/db/_tables/english.py
runtime-data/english/
```

**删除理由**：
1. **定位不符**：与核心"记忆训练"定位不符，是独立产品方向
2. **功能重复**：视频ASR + 句级翻译 + 打字练习，市场已有成熟工具
   - YouTube 自动字幕
   - Anki 听力卡片
   - Aboboo 听力训练
3. **维护成本高**：需要维护视频存储、ASR任务队列、进度管理
4. **无协同价值**：与记忆宫殿、间隔重复的协同性为零

**删除步骤**：
- [ ] 删除后端模块：`apps/api/src/memory_anki/modules/english/`
- [ ] 删除前端模块：`apps/web/src/features/english/`
- [ ] 删除数据表：`EnglishCourse`, `EnglishSentence`, `EnglishCourseProgress`, `EnglishGenerationTask`
- [ ] 删除运行时目录：`runtime-data/english/`
- [ ] 删除API路由：`english_router` 从 `main.py` 移除
- [ ] 删除仪表盘统计：`english_stats` 相关代码
- [ ] 更新文档：从 `PROJECT_DOCUMENTATION.md` 移除相关章节

**预计节省**：
- 代码：~3000行
- 数据表：4个
- API端点：~15个

### 3. 语音教练模块（voice_coach）

**涉及文件**：
```
apps/api/src/memory_anki/modules/voice_coach/
apps/web/src/features/voice-coach/
runtime-data/voice_coach/
```

**删除理由**：
1. **炫技而非刚需**：TTS事件播报是"锦上添花"，非核心功能

3. **效果有限**：对学习效果无实质帮助，反而可能干扰专注
4. **替代方案简单**：如需反馈，视觉反馈 + 简单音效即可

**删除步骤**：
- [ ] 删除后端模块：`apps/api/src/memory_anki/modules/voice_coach/`
- [ ] 删除前端模块：`apps/web/src/features/voice-coach/`
- [ ] 删除运行时目录：`runtime-data/voice_coach/`
- [ ] 删除API路由：`voice_coach_router` 从 `main.py` 移除
- [ ] 删除设置入口：从 `ProfilePage.tsx` 移除语音教练配置
- [ ] 保留简单音效：保留 `legacyWebAudio.ts` 中的基础音效播放

**预计节省**：
- 代码：~1500行
- API端点：~3个
- TTS缓存：动态大小

### 5. 时间记录模块（time_records）- 简化为自动统计

**当前功能**：
```
- 手动创建/编辑时间记录
- 软删除/恢复
- 遗留数据迁移
- 时区归一化
- 阈值过滤
- 批量删除
```

**问题**：
1. **反人性**：手动记录时长是反人性的设计
2. **功能过重**：可编辑、软删、迁移，功能过于复杂
3. **焦点错位**：应该关注"学习行为洞察"而非"时间记账"

**改造方案**：

**删除**：
- [ ] 删除手动创建/编辑对话框
- [ ] 删除软删除/恢复功能
- [ ] 删除遗留数据迁移
- [ ] 删除批量删除/选择功能

**保留并优化**：
- [ ] 自动记录学习时长（后台计时器）
- [ ] 只展示统计图表（趋势图、分布图）
- [ ] 不允许手动编辑（数据真实性）

**精简后的模型**：
```typescript
// 简化为纯统计表
interface StudySession {
  id: number
  session_type: 'edit' | 'practice' | 'review' | 'quiz'
  palace_id?: number
  duration_seconds: number
  started_at: timestamp
  ended_at: timestamp
  // 不再需要 is_deleted, device_id, notes 等字段
}
```

**预计节省**：
- 代码：~1000行
- 功能：5个复杂操作
- 认知负担：大幅降低

### 6. 多种复习模式 - 整合为单一最优策略

**当前复杂度**：
```
- 宫殿复习（review）
- 分段复习（segment_review）
- 迷你宫殿复习（mini_review）
- 批量分段复习（batch_segment_review）

- 艾宾浩斯算法
- SM2算法
- 自定义算法
- 算法迁移工具
```

**问题**：
1. **选择过多**：让用户选择等于没有设计
2. **粒度混乱**：三种复习粒度，用户不知道该选哪个
3. **算法暴露**：用户不需要理解艾宾浩斯 vs SM2

**整合方案**：

**删除**：
- [ ] 删除"分段复习"独立入口
- [ ] 删除"迷你宫殿复习"独立入口
- [ ] 删除算法选择器（review_mode 字段）
- [ ] 删除 SM2 算法实现
- [ ] 删除算法迁移工具

**统一为**：
```python
# 智能分块复习
class AdaptiveReviewStrategy:
    """
    根据宫殿大小和节点数量，自动决定复习粒度：
    - 小宫殿（<20节点）：整体复习
    - 中宫殿（20-100节点）：自动分段（基于层级）
    - 大宫殿（>100节点）：自动生成迷你宫殿
    
    统一使用增强版艾宾浩斯算法：
    - 遗忘曲线
    - 难度系数
    - 学习历史
    """
    
    def schedule_next_review(self, node, score):
        # 单一算法，无需用户选择
        pass
```

**用户视角**：
- 点击"开始复习" → 系统自动推荐今日复习内容
- 无需关心粒度、算法，专注于学习本身

**预计节省**：
- 代码：~3000行（多套复习流程）
- 数据表：6个（*ReviewSchedule, *ReviewLog的分段和迷你版本）
- API端点：~15个
- 认知负担：极大降低

### 7. 运行时 Supervisor - 删除

**涉及文件**：
```
apps/api/src/memory_anki/supervisor/
tools/start_supervisor.py
tools/runtime_supervisor.py
runtime-data/supervisor-state.json
```

**删除理由**：
1. **过度设计**：发布/代理守护、候选/当前版本切换，适合企业不适合个人
2. **复杂度高**：源指纹计算、发布流程、会话粘滞
3. **实际无用**：个人项目直接 `uvicorn` 启动即可

**删除步骤**：
- [ ] 删除 `supervisor/` 模块
- [ ] 删除 `tools/start_supervisor.py` 和 `tools/runtime_supervisor.py`
- [ ] 简化 `start.bat` 为直接启动 uvicorn
- [ ] 删除 runtime generation 版本控制

**替代方案**：
```bash
# start.bat 简化为
cd apps/api
python -m uvicorn --app-dir src memory_anki.app.main:app --port 8012

# 前端 dist 直接 serve（在 main.py 中）
app.mount("/", StaticFiles(directory=WEB_DIST_DIR, html=True))
```

**预计节省**：
- 代码：~2000行
- 复杂度：大幅降低

## ⚠️ 注意事项

1. **数据迁移**：
   - 删除前导出现有英语听力/阅读数据（如果有使用）
   - 时间记录保留历史数据，只删除手动编辑功能

2. **渐进式删除**：
   - 不要一次性删除所有模块
   - 建议顺序：Supervisor → 语音教练 → 英语模块 → 知识体系精简 → 复习模式整合

3. **用户通知**：
   - 虽然是个人项目，但建议在删除前确认自己不再依赖这些功能
   - 可以先"隐藏"入口，使用一周后再彻底删除代码
