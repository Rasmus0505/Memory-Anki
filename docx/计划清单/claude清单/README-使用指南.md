## 🎯 执行原则

### 1. 删除优先于新增
先做减法，再做加法。清理完债务，再建新功能。

### 2. 体验优先于功能
宁可功能少，也要体验好。不追求大而全。

### 3. 简单优先于完美
个人项目，灵活性比严格性重要。快速迭代。

### 4. 可见优先于隐藏
让学习进度和成就可视化，增强反馈感。

### 5. 渐进优先于重写
分阶段重构，保持可用状态，不要一次性大改。

## ⚠️ 注意事项

### 数据备份
**每次大改前**：
```bash
# 备份数据库
cp runtime-data/data/memory_palace.db backup/$(date +%Y%m%d).db

# 备份代码
git branch backup-before-refactor-$(date +%Y%m%d)
```

### 渐进式改变
- 不要一次性删除所有模块
- 先隐藏入口，测试1周，再删代码
- 保持主分支可用

### 用户通知
虽然是个人项目，但建议：
- 记录每次大改的原因
- 保留截图对比（改进前后）
- 写改进日志
