# 小宫殿与完成流程统一改造

## Summary
- 把现有“小宫殿”从“编辑页里的临时练习切片”升级为“父宫殿下的一级可复习单元”，拥有独立的练习进度、正式复习进度、排程、日志和书架展示。
- 所有练习页都补齐“练习 / 编辑”切换，交互对齐正式复习页；专项练习、分块练习、小宫殿练习在复习态看裁剪脑图，在编辑态切回父宫殿完整脑图。
- 所有会话页的右上角“完成”按钮先弹“未完成 / 已完成”决策层；`未完成` 只保存当前进度并留在当前页，`已完成` 才走现有“揭示剩余卡片 + 标红 + 选择推进到哪个复习节点 + 完成但仍需练习”的后续流程。
- 书架里的“开始复习”按钮改为整条可点击的复习进度条，显示当前复习阶段进度；小宫殿也使用同样样式，并纳入到期/练习统计。

## Public APIs / Types
- 扩展 `MiniPalaceSummary`，补齐和 `PalaceSegmentSummary` 同级的复习字段：`estimated_review_seconds`、`review_stage_total`、`review_stage_completed`、`review_stage_progress`、`stage_labels`、`review_stages`、`next_review_at`、`has_due_review`、`current_review_schedule_id`、`current_review_type`。
- 在宫殿列表/分组返回里增加 `mini_palaces`，让父宫殿卡片能直接渲染小宫殿子卡片；`palace_count` 仍保持“父宫殿数量”语义不变，但到期/练习统计改为聚合父宫殿和小宫殿。
- 新增小宫殿正式复习与进度接口，命名对齐现有分块接口：
  `GET /palace-mini-palaces/{id}` 返回练习用裁剪脑图与父宫殿信息。
  `GET /mini-review/session/{schedule_id}` 返回小宫殿正式复习会话。
  `POST /mini-review/session/{schedule_id}/submit` 提交小宫殿正式复习。
  `GET|PUT|DELETE /sessions/mini-practice/{mini_palace_id}/progress`。
  `GET|PUT|DELETE /sessions/mini-review/{schedule_id}/progress`。
  `PUT /palace-mini-palaces/{id}/review-progress` 供无活动排程时手动推进阶段。
- 扩展 `SessionProgressSnapshot.session_kind`，增加 `mini_practice`、`mini_review`；多块复习不加新服务端实体，未完成恢复走本地持久化键 `batch-review:{segmentIds}`。

## Implementation Changes
- 后端数据层：
  为小宫殿新增 review schedule / review log 表，结构对齐 `palace_segment_review_schedules` 和 `palace_segment_review_logs`。
  扩展 `session_progress`，增加 `mini_palace_id`、`mini_palace_review_schedule_id` 唯一索引。
  小宫殿创建时立即生成首轮排程；删除父宫殿级联删除小宫殿及其排程/日志/进度。
  父宫殿脑图结构变动后继续沿用现有 checkpoint 清洗逻辑；若小宫殿节点被清空，子卡片保留但进入不可复习空状态。
- 小宫殿编辑与选择：
  保留脑图上方“小宫殿”入口，现有新建流程继续使用左键单点、右键整棵子树、再次点击反选。
  将新建确认按钮改成明显的 `确认新建小宫殿`，保存成功后给出 toast。
  现有小宫殿增加“编辑节点”能力；书架子卡片点“编辑”时跳到父宫殿编辑页，并通过查询参数进入该小宫殿的预选节点编辑态，例如 `/palaces/:id/edit?miniPalaceId=123&miniPalaceMode=edit`。
  编辑态打开时自动载入该小宫殿已有节点为预选状态，确认后调用现有 `updateMiniPalaceApi` 更新 `node_uids`。
- 练习 / 编辑切换：
  去掉 `MindMapReviewFlow` 里“只有 review 才能 inline edit”的限制，改为只要传入 `editEditorState`、`onModeToggle`、`onEditEditorStateChange` 就允许切换。
  新增一个共享的 practice-session 容器，普通练习、专项练习、分块练习、小宫殿练习统一接入。
  普通练习的编辑态直接编辑当前宫殿完整脑图。
  专项练习、分块练习、小宫殿练习的复习态继续使用裁剪脑图，编辑态切到父宫殿完整脑图；退出编辑态后重新拉取裁剪会话数据并对 reveal/progress 做 sanitize。
- 完成按钮统一收尾：
  在 `MindMapReviewFlow` 的会话级“完成”按钮前新增 `CompletionDecisionDialog`。
  选 `未完成` 时不揭示剩余卡片、不改 completed，只立即持久化当前 snapshot，保留在当前页并提示“已保存进度，下次可继续”。
  选 `已完成` 时才调用现有 `finishFlow`，继续进入当前已有的 `StageSelectDialog` / 提交流程。
  正式复习页保持现有 `submitSession` 分支。
  练习页新增“完成即推进复习阶段”适配层：普通宫殿/专项练习使用父宫殿默认分块摘要，分块练习使用当前分块摘要，小宫殿练习使用小宫殿摘要；有 `current_review_schedule_id` 时走对应 submit API，没有活动排程时走对应 `review-progress` 调整 API。
  书架上的“标记已复习”这类快捷动作先不改，仍保持一键提交。
- 书架与宫殿卡片：
  在 `PalaceListCard` 里为父宫殿渲染一层小宫殿子卡片，视觉和当前普通宫殿卡片同构，但标题前缀显示所属父宫殿关系即可。
  小宫殿子卡片至少提供 `复习`、`练习`、`编辑` 入口；删除和重命名继续留在“小宫殿”管理面板。
  现有“开始复习”按钮改成整条可点击的阶段进度条组件，复用 `PalaceStageProgress` 的阶段填充逻辑与状态色；单宫殿、分块子卡片、小宫殿子卡片统一替换。
  进度条保留当前状态语义：到期用绿色强调，今天稍后用琥珀色，睡前/特殊类型沿用现有色彩提示；无可用排程时显示禁用态。
  学科书架顶层卡片不展示小宫殿明细，但其红点和统计要计入小宫殿的到期/练习状态。

## Test Plan
- 后端单元/API：
  创建小宫殿即生成独立排程，父宫殿与小宫殿进度互不影响。
  小宫殿 CRUD 后 `mini_palaces` 列表和 grouped/shelf 聚合返回正确。
  小宫殿节点被父脑图删除后自动清洗，空小宫殿不会产生非法会话。
  小宫殿 practice/review progress 的保存、恢复、清除互不串号。
  小宫殿 review submit 与 manual progress adjust 都能正确推进阶段并写日志。
- 前端单测：
  普通练习、专项练习、分块练习都出现“练习 / 编辑”切换，切回后会重新生成对应裁剪脑图。
  完成按钮点击后先弹“未完成 / 已完成”；`未完成` 不 reveal、不 completed、会保存 snapshot；`已完成` 才进入现有阶段选择。
  小宫殿现有 reveal-flow 单测继续保留，并补“编辑已有小宫殿时预选节点加载、左右键反选、保存成功 toast”。
  `PalaceListCard` 能渲染 mini palace 子卡片；点击可进入练习/复习/编辑；可点击进度条替代原“开始复习”按钮。
  多块复习的 `未完成` 能从本地持久化恢复。
- 集成回归：
  复习页已有 inline edit、StageSelectDialog、完成但仍需练习不回退。
  书架统计在含小宫殿和不含小宫殿时都正确，`palace_count` 显示仍代表父宫殿数量。
  小宫殿与父宫殿同时存在到期任务时，二者卡片与统计都能独立展示。

## Assumptions / Defaults
- 小宫殿卡片的“编辑”不是独立页面，而是跳回父宫殿编辑页并直接进入该小宫殿的节点重选模式。
- `未完成` 的语义是“显式保存当前断点”，保存后继续停留当前页，不结束计时、不自动退出。
- “完成按钮统一改造”只覆盖会话页右上角的完成入口，不改书架里的快捷完成按钮。
- 练习页的“练习 / 编辑”范围覆盖普通练习、专项练习、分块练习，以及后续新增的小宫殿练习页。
- 小宫殿与父宫殿允许节点重叠，继续沿用当前允许重叠的规则。
