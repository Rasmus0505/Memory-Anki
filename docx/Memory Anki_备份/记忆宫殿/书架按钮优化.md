
# 宫殿卡片操作、复习样式及小宫殿（Mini-Palace）复习归属权

## 摘要
- 将单分段的主宫殿复习操作移回卡片顶部操作区，使其不再作为独立的通栏显示。
- 加强复习状态的样式：`开始复习` 使用更深绿色的视觉效果；“今日需复习但未到期”使用鲜明的黄色按钮；复习进度条切换至冷色调的进度专用配色，使其不再与行动呼吁（CTA）混淆。
- 将原本独立的删除图标替换为包含`配置`和`删除`选项的“三点式”溢出菜单。
- 增加宫殿级的小宫殿复习归属模式，提供 `independent`（独立复习）和 `mini_only`（小宫殿接管复习）两个选项。在本规范中，`mini_only` 意味着只要宫殿内至少包含一个“小宫殿”，小宫殿将完全接管正式复习流程。
- 在所有与完成相关的对话框中显示`本次耗时`，确保用户在“已完成”和“未完成”路径下看到相同的用时统计。

## 关键变更
- **宫殿卡片 UI**
  - 保持`练习`和`编辑`按钮可见。
  - 用 `MoreHorizontal`（更多）溢出菜单触发器替换原有的独立删除按钮。
  - 溢出菜单项：`配置`用于打开对话框，`删除`保持现有的破坏性确认流程。
  - 对于单分段宫殿，将主复习 CTA 渲染在顶部操作集群中，而非全宽内联栏。
  - 更新复习 CTA 样式：
    - `due_now`（立即复习）：更深、更具对比度的翡翠绿填充按钮。
    - `due_later_today`（今日后续）：清晰的黄色/琥珀色填充按钮，而非淡薄荷绿。
    - `sleep`（休眠）：保持蓝色系。
  - 更新 `PalaceStageProgress`（宫殿阶段进度）的颜色，使用针对进度的冷色调填充已完成节点，未完成节点保持灰色。
- **宫殿 vs. 小宫殿的复习归属权**
  - 在宫殿对象中增加字段 `mini_review_mode: 'independent' | 'mini_only'`。
  - 在宫殿列表/详情的数据载体（Payload）中暴露该字段，并增加专门的更新接口，如 `PUT /palaces/{id}/mini-review-mode`。
  - 配置对话框选项：
    - `独立复习`：主宫殿与小宫殿各自保留正式复习。
    - `小宫殿接管复习`：只进行小宫殿的正式复习。
  - 当宫殿内至少存在一个小宫殿时，`mini_only` 模式的逻辑：
    - 在列表卡片上隐藏或禁用主宫殿的正式复习 CTA 及进度条；
    - 列表/架上的“立即复习”和“今日后续”状态仅根据小宫殿的排期导出；
    - 在有效队列/统计呈现中排除主宫殿的正式复习排期，但不删除已存储的记录；
    - 保持宫殿练习/编辑及小宫殿练习/复习的行为不变。
- **完成对话框**
  - 将当前有效的会话时长传入 `CompletionDecisionDialog` 和 `StageSelectDialog`。
  - 在两个对话框中显示紧凑的 `本次耗时：...` 摘要，确保完成和未完成路径在提交前拥有相同的计时上下文。

## 公共 API / 类型
- 在 `/palaces`、`/palaces/grouped` 等接口返回的宫殿相关契约，以及本页面消费的任何其他宫殿摘要数据中增加 `mini_review_mode`。
- 为新的宫殿复习模式更新接口增加前端 API 辅助函数。
- 扩展完成对话框的 Props，以接收原始或格式化后的时长数据。

## 测试计划
- **UI 测试**
  - 单分段宫殿卡片不再渲染独立的通栏复习栏/按钮。
  - “今日后续”复习按钮使用更醒目的黄色样式。
  - 删除操作不再是独立图标，且可通过溢出菜单访问。
  - 配置对话框显示两种复习归属模式，并能持久化所选模式。
- **复习流程测试**
  - `完成本次练习/复习` 显示`本次耗时`。
  - 正式复习的`选择复习进度`界面也显示相同的时长。
  - 未完成的复习路径依然会记录时间并按原逻辑重置计时器。
- **后端测试**
  - 宫殿 JSON 数据包含 `mini_review_mode`。
  - 当 `mini_only` 模式下的宫殿拥有小宫殿时，会抑制主宫殿的有效复习状态，并使待复习计数来源于小宫殿排期。
  - 切换模式不会删除现有的宫殿复习排期或日志。

## 假设
- `mini_only` 仅在宫殿已至少包含一个小宫殿时才生效；否则保留所选模式，但行为与 `independent` 保持一致。
- 切换至 `mini_only` 是可逆的，不会重建或清除历史的主宫殿复习数据。
- 新的溢出菜单包含`配置`和`删除`；`编辑`作为可见的铅笔图标保留在菜单外。
