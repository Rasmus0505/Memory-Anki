# 功能层清单

> 独立审视结论，不依赖仓库内既有文档框架。按"该删除/降级""该优化""该新增"三类组织，每项标注优先级（P0 最高）。

---

## 一、该删除 / 降级

### [P2] 英语听力与英语阅读的平行实现
- **现状**：`english`、`english_reading` 各自维护一套 session、progress、AI 生成任务、进度校准逻辑，和宫殿/复习主线完全独立。
- **目标**：不删除功能本身，但把它们降级为"内容源适配器"，接入统一的会话/进度/AI 任务基础设施（具体方案见架构层清单）。
- **理由（Why）**：三份并行实现意味着三份 bug、三份测试、三份未来要改的地方。英语能力不该是独立王国，而应该是这台学习引擎上的一种内容形态。
- **验收标准**：英语听力/阅读的"做一道题/读一句话"这类动作，最终落库路径和宫殿复习共用同一张会话表结构，而不是各自的 progress 表。

### [P2] 时间记录的 legacy 迁移脚手架
- **现状**：`session-records-legacy.ts`、后端 `import-legacy` 端点、`normalize-timezones` 端点长期挂在代码里，服务于早已完成的一次性数据迁移。
- **目标**：确认两台设备的历史数据都已迁移完成后，彻底删除这些兼容代码和端点。
- **理由（Why）**：个人产品只有你一个用户、两台受控设备，没有"未知客户端还在用旧格式"的风险，没必要长期为一次性迁移买单。
- **验收标准**：搜索仓库不再出现 `legacy` 相关的时间记录代码；两台设备真实数据核对无误后再删除，删除前做一次全量备份。

### [P3] 一次性脚本与审计产物
- **现状**：`.audit/`（`1000-quiz`、`sync-rev-000002` 等目录）、`tools/ocr/render/merge_1000_*.py` 是历史批量生产/审计的产物，长期留在仓库里。
- **目标**：迁出仓库（本地归档）或直接删除，只保留脚本本身如果还有复用价值。
- **理由（Why）**：不影响功能运行，但会持续污染"这是不是还在用"的判断成本，个人项目也要维护代码库的可信度。
- **验收标准**：`git ls-files` 中不再出现这些一次性产物目录；如果脚本还有用，移到 `tools/` 下并写清楚用途。

## 二、该优化（已存在但达不到应有水准）

### [P0] 随心模式（"刷抖音式学习"）投入严重不足于设计意图
- **现状**：后端 `freestyle` 模块只有 3 个 application 文件 + 1 个 router，是全仓库最单薄的模块之一；但你在笔记里已经明确设计过一整套体验——仿抖音连续刷题、刷题范围可选（按宫殿完整刷完/随机/指定宫殿/待复习宫殿）、做题时可弹窗查看宫殿不打断心流、宫殿浮窗要能像计时器一样部分滑出屏幕并支持"查看模式/翻卡模式"切换。这些设计目前基本没有对应到位的实现。
- **目标**：把随心模式做成真正的"心流训练空间"——連续出题不打断、范围可配置、宫殿速览浮窗与翻卡浮窗到位、强反馈驱动"想继续刷下去"的欲望。
- **理由（Why）**：这是你自己反复设计、最贴近"易于进入心流、高反馈"这个产品北极星的功能，但代码实现明显滞后于设计意图，是当前功能层里性价比最高的投入方向。
- **验收标准**：能配置"按宫殿顺序/随机/指定宫殿/待复习宫殿"四种刷题范围；做题过程中可以呼出宫殿速览浮窗且不中断当前刷题状态和计时；浮窗支持"查看/翻卡"切换且复用现有复习翻卡逻辑（不需要新后端接口）。

### [P0] 反馈系统未达标，配置修改不生效
- **现状**：你在笔记里明确指出个人中心的反馈配置"当前版本未达标"，形容词类的调整（轻体、电影感等）修改后并不生效；音量设计表现良好可保留；烟花特效（`celebrationEngine.ts` 基于 canvas-confetti）已经定义了 random direction / realistic / fireworks / stars / school pride 五种类型，但前端缺少直观预览，点击对应板块应立即在界面里演示效果而不依赖右侧实时预览台；烟花触发位置也需要固定为"从网站边缘喷出、层级最高"，而不是屏幕中央或局部区域。
- **目标**：整体重建反馈配置页的布局与后端逻辑，做到"点哪个类型、立刻在当前界面看到那个类型的真实效果"，配置修改要真正生效并持久化。
- **理由（Why）**：反馈系统是"高反馈学习"这个产品承诺的具体落地，如果配置本身不可信、不直观，用户对整个反馈体系的信任都会打折扣。
- **验收标准**：五种烟花类型点击后 1 秒内在当前页面触发一次真实预览；所有反馈相关配置修改后刷新/重进应用仍然生效；烟花统一从视口边缘发射、显示层级高于所有其他 UI。

### [P0] 页面状态持久化不完整，存在阻断性 bug
- **现状**：路由驻留缓存（LRU，上限 12 条）理论上应该保留页面状态，但你实测发现从书架打开宫殿、切到其他侧边栏页面再切回来，看到的是书架而不是刚才打开的宫殿；宫殿编辑页甚至会直接打不开。AI 生成等异步任务在切页后状态是否保留也不确定。
- **目标**：修复驻留缓存的实际行为缺口，确保"切走再切回"能精确恢复到离开时的页面和状态（不只是同级路由，包括宫殿详情这类带参数的二级状态）；同时排查并修复宫殿编辑页打不开的阻断性问题。
- **理由（Why）**：这是当前用户体验里最直接、最频繁触发的挫败点——一个号称支持状态驻留的机制，实际表现却是"随时可能丢状态"，这比完全不做驻留缓存更让人困惑，因为不可预测。
- **验收标准**：从书架进入宫殿详情，切到任意其他侧边栏页面，再切回来，看到的必须是宫殿详情而非书架；宫殿编辑页在所有已知触发路径下都能正常打开；写一条端到端回归测试覆盖这个具体场景，防止再次回退。

### [P1] 全局悬浮计时器局限在产品页面内
- **现状**：`desktop-timer` 已经是一个独立的 Electron 置顶悬浮窗应用，能在主网页最小化时依然悬浮显示；但目前只服务于"学习/休息"这一组场景，没有覆盖"我要切换到非本产品的页面/应用（比如看会儿电视）时也能帮我计时提醒"这个你明确提出的需求。
- **目标**：扩展悬浮计时器的场景，支持"离开本产品去做别的事情"时也能设定一个倒计时/提醒，到点时通过悬浮窗、系统通知或音效提醒你回来，而不只是服务本产品内的学习/休息切换。
- **理由（Why）**：你已经拥有系统级悬浮窗的技术基础（Electron alwaysOnTop 窗口），只是场景设计上还局限于"学习页面内的计时器"，稍加扩展就能覆盖"防止看电视看忘了时间"这类真实生活场景，投入产出比很高。
- **验收标准**：可以在离开本产品前设置一个自定义时长的"离开提醒"；计时器悬浮窗在切到任意其他应用（包括全屏视频）时依然可见或能弹出系统级提醒；提醒方式（视觉/声音）可配置。

### [P1] 手动导入试题对非标准格式文件不友好
- **现状**：你有真实的手动整理试题材料（如 `waijiao`、`zhongjiao` 目录下的文件）需要导入现有宫殿题库，但这些文件格式不一定符合系统的解析要求；当前"AI 转试题"面板没有针对这种情况的引导。
- **目标**：在 AI 转试题面板里增加"手动导入文件"入口，当识别到文件格式不规范时，给出一段可直接复制发给 AI 的提示词，引导用户先用 AI 把内容改造成系统能识别的格式，再回来导入；导入时要做去重（不重复题目）并确保所有题型都能被正确导入。
- **理由（Why）**：现有格式化的试题材料是现成的学习资产，不能因为格式不完全匹配就沦为"死文件"，一段引导性提示词的实现成本很低，收益是直接可用的题库内容。
- **验收标准**：上传不规范格式文件时，界面出现明确的"格式不符合要求"提示和可复制的引导提示词；重新上传修正后的文件能正确解析出所有题型；导入时对已存在的相同题目自动跳过不重复创建。

### [P1] 同步体验：冲突处理需要人工翻文件夹
- **现状**：详见架构层清单"同步架构"条目——当前一旦两台设备在未同步情况下都发生了修改，会直接阻断启动，需要手动去 `conflicts/` 目录处理冲突快照。
- **目标**：至少先做一个应用内的"合并提示/选择保留哪份"轻量引导 UI，取代纯文件系统层面的手工排查。
- **理由（Why）**：这是当前最容易让你"卡在无法进入应用"这一步的场景，即便暂不做架构级重做，也应该先把交互体验做人性化。
- **验收标准**：出现冲突时，应用内能直接展示"检测到冲突，两份快照的关键差异是什么，选择保留哪一份"，不需要手动打开文件夹核对文件内容。

## 三、该新增（当前完全不存在的能力）

### [P0] 统一薄弱点 / 错题中心
- **现状**：测验的错题、复习中反复"翻车"的节点（红标）、阅读校准中的失误，是三套互不相通的数据，没有任何地方能一次性看到"我到底哪里不会"。
- **目标**：新增一个跨内容类型的薄弱点聚合视图，把测验错题、复习红标节点、阅读校准失误统一呈现，并作为随心模式/正式复习之外的一个独立、高优先级的练习入口。
- **理由（Why）**：这是间隔重复类产品里投入产出比最高的功能——比起被动等待固定周期的复习轮次，主动暴露"哪里薄弱"能立刻转化成有针对性的练习动作，直接对应"高反馈"这个产品目标。
- **验收标准**：能在一个页面看到跨宫殿/跨模块的薄弱点列表，每条能追溯到具体节点/题目/材料，点击可直接进入针对性练习。

### [P1] 遗忘曲线 / 高危知识点预测
- **现状**：仪表盘只做"今日复习量、时长统计"这类回顾型指标，没有任何前瞻性判断。
- **目标**：结合调度算法（见架构层 FSRS 迁移），做出"哪些节点大概率会被遗忘""哪些主题保留率在下降"的预测性提示。
- **理由（Why）**：真正好的间隔重复系统价值在于"提前告诉你什么会忘"，而不只是"事后统计你复习了多少"，这是把调度算法的价值真正兑现给用户可见的一步。
- **验收标准**：仪表盘能列出至少 3-5 个"近期高遗忘风险"的节点或主题，并给出简短理由（如"距上次复习已超预期间隔""近三次复习正确率下降"）。

### [P2] AI 学习教练升级为可对话式
- **现状**：`voice_coach` 只做场景化 TTS 单向播报（开始、里程碑、完成等固定文案）。
- **目标**：在错题/复习节点上增加一个轻量对话入口，可以临时问"换个方式讲讲这个""帮我编个例子/类比记住它"，而不只是被动听播报。
- **理由（Why）**：已经接入了通用 LLM 能力，播报这种单向输出没有用满 AI 的潜力；一个不打断当前流程的轻量问答，能显著提升"卡住时能不能立刻获得帮助"的体验。
- **验收标准**：在复习或做题界面上，能对当前节点/题目发起一次简短提问并得到即时回答，且不需要跳出当前页面或打断计时。

### [P1] 全局撤销 / 回收站
- **现状**：只有宫殿脑图有版本历史（`PalaceVersion`），其余删除操作（测验题、章节、附件、迷你宫殿等）都是硬删除，删错了没有找回的办法。
- **目标**：把"软删除 + 一段时间内可恢复"作为所有可删除实体的默认行为，而不是宫殿脑图独有的特权。
- **理由（Why）**：个人产品没有人替你兜底，一次手滑删除且没有恢复渠道，代价是真实数据损失；这个能力的实现成本远低于它能避免的心理负担和实际损失。
- **验收标准**：题目、章节、附件、迷你宫殿删除后进入"最近删除"，保留至少 7-14 天可一键恢复，超期自动清理。

### [P2] 全局速记捕获
- **现状**：学习过程中如果突然想到一个念头、一道临时题、一个疑问，没有不打断当前专注状态就能记下来的入口。
- **目标**：新增一个全局快捷键触发的"速记收件箱"，随手记录一句话，之后再统一分诊归类进宫殿或题库。
- **理由（Why）**：专注计时和当前任务的存在，反而容易让学习中的灵光一现被压下去或遗忘；低摩擦捕获是心流保护的一部分。
- **验收标准**：任意页面下用固定快捷键弹出一个极简输入框记录一条速记，不中断当前计时/任务状态；速记进入一个统一收件箱，之后可以手动转化为宫殿节点或题目。

### [P3] 语料语义检索
- **现状**：已经积累了 PDF、脑图、笔记等结构化内容，但没有任何"对自己的知识库提问"的能力。
- **目标**：提供一个轻量检索入口，能基于已有材料做语义搜索或简单问答。
- **理由（Why）**：内容摄入管道统一之后，这是顺带能做的低成本高附加值能力，不需要复杂 RAG 架构，SQLite FTS5 + 简单向量索引即可起步。
- **验收标准**：能对一个具体问题，检索出相关的宫殿节点/题目/材料原文片段作为参考。
