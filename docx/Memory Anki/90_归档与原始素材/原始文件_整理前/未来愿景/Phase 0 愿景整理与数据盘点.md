# Phase 0 愿景整理与数据盘点

> 上级文档：[[Memory Anki 未来架构愿景总纲]]

---

## 目标

为未来重塑建立清楚地图。

这一阶段不急着做新功能，而是把现有系统、数据和用户流程盘清楚。它是后续事件流、今日训练台和 AI 工作流重塑的准备期。

---

## 核心问题

Phase 0 要回答：

- 当前有哪些模块？
- 每个模块拥有哪些数据？
- 哪些数据是“状态”，哪些数据是“事件”，哪些数据可以变成“信号”？
- 当前用户从材料到训练的路径是什么？
- 今日训练台需要从哪些现有数据源取信息？
- 现有 AI 调用场景有哪些？
- 哪些数据未来需要迁移？

---

## 交付物

### 1. 当前模块与数据表盘点

需要盘点：

- `knowledge`
- `palaces`
- `palace_quiz`
- `reviews`
- `sessions`
- `time_records`
- `english`
- `english_reading`
- `settings`
- `backups`
- `dashboard`
- `freestyle`

每个模块记录：

- 数据 owner。
- API owner。
- UI owner。
- 核心表。
- 核心业务流程。
- 与其他模块的关系。

### 2. 当前用户流程图

至少画出这些流程：

- 新建宫殿。
- 编辑宫殿。
- 导入图片/PDF 到脑图。
- 生成题库。
- 做题。
- 正式复习。
- 分段复习。
- mini-palace 练习。
- 英语听力。
- 英语阅读。
- 随心模式。
- 时间记录和仪表盘。

### 3. 学习事件类型清单

初步列出未来要支持的事件：

- 复习完成。
- 复习失败。
- 练习完成。
- 题目作答。
- 简答题点评。
- 节点红标。
- 英语句子完成。
- 阅读完成。
- AI 任务完成。
- AI 输出被应用。
- 会话开始、暂停、中断、完成。

### 4. 现有数据到 LearningEvent 的映射表

示例：

| 现有数据 | 未来事件 |
|---|---|
| ReviewLog | `review_completed` |
| TimeRecord | `session_completed` 或 `study_duration_recorded` |
| PalaceQuizQuestion attempt | `quiz_answered` |
| EnglishReadingSession | `english_reading_completed` |
| ExternalAiCallLog | `ai_job_completed` 的证据 |

### 5. 今日训练台信息源清单

今日训练台至少需要：

- 待复习。
- 逾期复习。
- 需练习宫殿。
- 红标节点。
- 错题。
- 未完成英语课程。
- 未完成英语阅读材料。
- 未确认 AI 草稿。
- 最近导入材料。

### 6. AI 调用场景清单

需要盘点：

- 图片转脑图。
- PDF 转脑图。
- 文本预览。
- 题库生成。
- mini-palace 分类。
- 简答题点评。
- 英语听力生成。
- 英语阅读难度生成。
- 语音教练。

### 7. 宫殿质量指标草案

初步指标：

- 节点数量。
- 空节点比例。
- 长节点比例。
- 层级深度。
- segment 覆盖。
- mini-palace 覆盖。
- 题库覆盖率。
- 错题密度。
- 红标节点数量。

---

## 完成标准

- 能清楚说明每个现有模块未来归属。
- 能清楚说明哪些数据是状态，哪些数据是事件，哪些数据是信号。
- 能列出第一版 LearningEvent 类型。
- 能列出今日训练台第一版数据源。
- 能列出未来迁移风险点。
