# 模型分类+硅基

> 原始路径：大模型调用\模型分类+硅基.md

将个人主页的模型配置整理下，现在的太复杂了，场景很多不方便，以后模型的管理按模型本身类型分类
比如：大语言、VL、翻译、ASR等等
额外提示：大语言模型要额外备注是否有视觉能力，例如xx（有/无视觉）

按这样分类，然后鼠标悬浮可以看见该类模型应用到哪些具体场景，方便我判断该场景是不是真的要用这个模型，注意这个场景一定要很细致，防止程序在我不知情的情况下调用了较贵的大模型！！！！！同时可以针对场景进行单独配置模型
如果个人主页修改了模型配置，对应的场景相关前端也要更改模型名称方便我判断我的模型更改是否真的生效，例如前端总是显示vl-flash不变化。导致我在这种情况无法判断是否真的更改
关于模型的配置一定要做好，杜绝前端修改但后端实际调用的模型却不是的情况，支持删除模型功能
