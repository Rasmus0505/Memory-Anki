## 1. 产品定位

Memory Anki 是一个本地优先的个人学习训练系统。

它不是普通笔记库，也不是一堆 AI 生成按钮，而是把学习材料稳定转成可训练内容，并持续推动用户进入低负担、高反馈、可复盘的训练闭环。

核心闭环：

```text
学习材料 -> AI 结构化 -> 记忆宫殿 -> 题库 -> 复习 / 练习 -> 反馈 -> 下一步训练
```

产品边界：

- 只服务个人电脑端学习，不优先做移动端或公众平台。
- 所有功能都要回到训练闭环，不能继续长成互相割裂的模块。
- AI 是助教，不是黑箱主人；模型、参数、提示词、调用成本和生成记录都要透明。
- 数据本地优先，学习记录、宫殿、题库、计时、复习进度必须可靠保存。

北极星：

- 打开应用后，用户知道现在最值得训练什么。
- 用户能把资料低成本变成宫殿和题库。
- 训练过程中切页面、开浮窗、查词、做题、计时，都不丢状态。
- 系统能根据错题、薄弱点、复习表现和时间记录给出下一步行动。

## 2. 训练闭环

### 记忆宫殿与小宫殿

- 记忆宫殿是主要知识结构。
- 小宫殿是父宫殿下的独立训练单元，不是临时切片。
- 小宫殿应拥有独立练习、正式复习、排程、日志、书架展示。
- 小宫殿支持新建、编辑节点、增加节点、减少节点。
- 小宫殿默认名称使用用户选中的第一个节点，并保留根节点上下文。
- 普通宫殿、小宫殿、练习、复习共用同一套翻卡逻辑。
- 小宫殿不再默认自动翻开未标记节点；用户悬停节点后按空格，才沿该节点向下揭示，直到碰到标记节点。
- 多个并列分支要同时流通，不能只处理第一个分支。
- 所有会话页点击完成时，先选择“未完成 / 已完成”。
- “未完成”只保存断点；“已完成”才揭示剩余、标红、选择推进阶段。

### 随心模式

- 正式名称统一为“随心模式”，旧称“刷抖音式学习”不再作为主术语。
- 目标是像刷短视频一样连续刷题，但服务于学习。
- 支持全部、待复习、需练习、指定宫殿等范围。
- 支持随机、顺序、完整宫殿刷完再随机。
- “完整宫殿刷完”必须按父宫殿分组，不能被小宫殿拆散。
- 做题中可打开记忆宫殿浮窗，支持查看模式和翻卡模式。
- 浮窗拖拽、折叠、滑出屏幕边缘、切换模式，都不能重置题目、进度、连击或计时。

### 计时器与反馈

- 计时器是训练闭环的一部分，不只是小组件。
- 页面内全局可见，支持拖拽、调整大小、折叠为胶囊。
- 胶囊也可拖动，并能稳定恢复为卡片。
- 一级时间是总目标，显示为正计时累计/目标时长。
- 二级时间是小间隔，作为最显眼数字，引导用户关注小目标。
- 切换页面或短暂离开时，计时显示不中断。
- 数据入库按场景记录，但前台训练体验保持连续。
- 闲置阈值内返回学习，应合并为同一条时间记录。
- 所有视觉、音效、烟花反馈走统一反馈系统。
- 反馈配置必须真实生效，并支持点击即预览。
- 烟花从整个视口边缘喷发，层级最高，不被局部容器裁剪。
- 有对错判定的场景都应有即时反馈。

## 3. 内容生产与题库

目标是把“上传资料 -> AI 识别 -> 结构化 -> 预览确认 -> 写入宫殿/题库 -> 进入训练”做成稳定流水线。

### 材料来源

- PDF、图片、Markdown、TXT、JSON 都可以作为输入。
- PDF 可长期保存为材料来源，不应因为取消预览页而失去价值。
- 题目资料和答案/解析资料可以来自不同文件，用户能标注角色。
- 对非标准文本，AI 先整理成程序可识别的题库结构。

### AI 生成流程

- 生成前可选择模型、参数和提示词。
- 配置记忆上次选择，并提供重置。
- 生成中显示流式进度，避免空白等待。
- 生成后先预览，用户确认后再写入正式数据。
- 每次生成记录输入材料、模型、参数、提示词、AI 返回和写入结果。

### 题库能力

- 每个宫殿和小宫殿都可以有关联题库。
- 支持选择题、判断题、填空题、连线题、排序题、归类题、简答题。
- 选择题点击后立即判定，错项变红，正确项变绿，解析自动展开。
- 简答题提交后显示参考答案和解析，可请求 AI 点评。
- 所有题目标注来源：PDF、AI 生成、手动导入、复习生成等。
- 题库写入前自动去重，重复题只保留一个。
- 题目、答案、解析、选项必须结构化保存。
- 小宫殿可以从父宫殿题库中筛选相关题。
- 生成题目前可勾选“按小宫殿分类保存”。
- 选择大章节生成时，可以一次覆盖其下所有小宫殿。
- 如 VL 模型无法稳定匹配题目和答案，可使用文本模型做二次配对。

## 4. 学习体验

### 页面状态与断点续跑

- 页面切换不能重置上一个页面正在进行的任务。
- 宫殿详情、宫殿编辑、AI 生成、随心模式、英语阅读、英语听力都要保存关键状态。
- 从书架进入宫殿后，切到其他页面再回来，应仍在该宫殿上下文。
- AI 生成任务继续运行，页面回来后重新订阅进度或结果。
- 宫殿编辑页打不开、切回变书架、生成进度丢失，都是 P0 阻断问题。

### 英语阅读

- 英语阅读是“i+1 材料生产 + 阅读训练”的内容适配器，不是独立产品。
- 本地 CEFR 词典优先；未识别词再交给 AI 做原型还原和等级评估。
- AI 返回 1-3 个候选原型后，再查本地词典；命中则以词典为准，否则用 AI 等级。
- AI 无有效结果时保守跳过，不能默认标成某等级。
- 改写原则：原意优先，难度第二；不能为了简单而篡改意思。
- 绿色表示原文已处于 i+1；黄色表示低于 i+1 且实际升级；红色表示高于 i+1 且实际降阶；黑色表示其余词。
- Hover 显示最终 CEFR、原文、可选还原词、来源“词典/AI”。
- 查词弹窗只保留美式音标、美式发音、中文释义、词性等必要信息。
- 弹窗可滚动、可置顶、可拖动，查下个词不重置位置。
- 长按选择句子后手动确认翻译，翻译弹窗显示原文和译文。

### 英语听力

- 英语听力是听写训练适配器，不应维护割裂的学习系统。
- 播完一句后暂停，等待用户完成本句拼写。
- 本句全部单词出现后，翻译区立刻更新为本句翻译。
- 完成后重播本句，并无停顿衔接下一句，然后暂停。
- 页面一屏内应看见视频、拼写区和翻译区。
- 视频按比例显示，不拉伸，不保留大量无意义黑边。
- 拼写区从字母方框改为下划线，减少视觉负担。
- 音效柔和可调，包括键盘音、答对/错提示、完成句子音。

## 5. AI 模型与运行架构

### AI 模型管理

- 按模型类型管理，而不是按零散场景堆叠。
- 类型包括：大语言模型、VL/OCR、翻译、ASR、自定义模型。
- 大语言模型需标注是否有视觉能力。
- 所有 AI 场景都要显示当前实际调用模型。
- 支持全局默认模型和场景级覆盖。
- 场景级覆盖要有明显标记，说明它不同于全局默认。
- 生成前可以临时选择模型、参数和提示词，并记住上次配置。
- 支持删除内置和自定义模型；删除后相关场景回退到有效默认。
- 昂贵模型调用必须透明，不能静默发生。
- 主文档不保存 API key；密钥只留在外部原始归档。

统一 AI 工作流：

```text
选择材料/场景 -> 选择模型与参数 -> 生成中可观察 -> 输出可预览 -> 用户确认 -> 写入正式数据 -> 记录调用轨迹
```

### 本地运行与同步

- 稳定学习环境和开发环境分离。
- 日常学习使用稳定版，不受后台改代码影响。
- 开发 worktree 可频繁改动和测试。
- 数据库从代码目录中解耦，放入受控数据目录。
- 启动方式要稳定，不因更新代码反复损坏。
- 网页刷新即可切换新版本，不要求用户理解复杂服务状态。
- 当前目标是两台电脑通过百度网盘同步空间使用同一套学习数据。
- 同步重点是学习数据，不是构建产物和临时缓存。
- 需要明确备份、迁移、冲突处理、软删除和版本历史。

### 未来架构

- 前端统一：路由状态、脑图工具栏、反馈系统、计时器、长任务订阅。
- 后端统一：学习事件、AI 任务、题库、复习调度、材料来源、日志。
- 数据统一：材料、宫殿、题目、复习、练习、时间、反馈都能形成 LearningEvent。
- 训练编排统一：系统根据事件生成薄弱点、错题、NextAction 和 TrainingPlan。

## 6. 路线图

### Phase 0：盘点与收敛

- 盘点现有模块、数据表、AI 调用、复习流程、题库流程、时间记录。
- 清理重复文档和过时描述。
- 建立 LearningEvent、KnowledgeSource、AiJobTrace 等核心对象草案。
- 确认哪些功能要删除、降级、合并。

### Phase 1：今日训练台与事件底座

- 建立今日训练台。
- 打通待复习、错题、随心模式、英语热身、计时器。
- 页面状态断点续跑作为 P0。
- 训练过程统一记录 LearningEvent。

### Phase 2：内容生产流水线

- PDF/图片/文本统一进入材料库。
- AI 生成可配置、可观察、可预览、可确认。
- 宫殿和题库写入前可审查。
- 题库去重、题目答案配对、小宫殿归属作为重点。

### Phase 3：训练计划与复盘

- NextAction 升级为 TrainingPlan。
- 建立薄弱点/错题中心。
- 做题、复习、英语、计时统一进入复盘。
- AI 生成复盘报告，但必须可追溯数据来源。

### Phase 4：表现驱动调度与结构演化

- 从固定间隔升级到表现驱动调度。
- 对高风险知识点提前提醒。
- 给出宫殿拆分、合并、重构建议。
- 支持学习策略实验记录。

## 7. 当前优先级

### P0

1. 页面状态与断点续跑：切页面不丢进度，宫殿编辑页稳定打开。
2. 反馈系统重建：配置真实生效，点击即预览，统一复用。
3. 随心模式收尾：完整宫殿分组、宫殿浮窗、翻卡模式、不中断计时。
4. 题库生成可靠性：PDF/文本生成、题目答案配对、去重、来源追踪。
5. 小宫殿训练单元化：独立复习/练习/排程/日志/书架展示。

### P1

1. AI 模型管理：全局默认、场景覆盖、生成前配置、调用透明。
2. 计时器连续记录：闲置阈值内合并记录，全局悬浮稳定。
3. 英语阅读 i+1 链路：词典优先、AI 兜底、生成记录、查词弹窗。
4. 英语听力沉浸页：播放逻辑、布局、音效、拼写区简化。
5. 本地运行与同步：稳定版/开发版分离，数据目录解耦。

## 8. 开工判断标准

所有新任务开工前都问三件事：

1. 它是否进入“材料 -> 训练 -> 反馈 -> 下一步”的闭环？
2. 它是否减少用户学习中断，而不是增加配置负担？
3. 它是否复用统一事件、AI、计时、反馈和状态机制？
