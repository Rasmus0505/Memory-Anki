# 记忆宫殿与小宫殿

## 背景

记忆宫殿是 Memory Anki 的核心学习载体。原始需求集中在脑图工具栏复用、宫殿卡片样式、小宫殿创建与编辑、复习/练习完成流程统一、翻卡逻辑复用等方向。

## 目标体验

- 用户在普通宫殿、小宫殿、练习、复习、编辑等场景里看到一致的脑图工具栏。
- 小宫殿不是临时切片，而是父宫殿下可独立练习、复习、排程、统计的学习单元。
- 完成按钮不直接结束流程，而是先让用户选择“未完成 / 已完成”，避免误结束。
- 书架卡片能清楚展示父宫殿和小宫殿的复习状态、练习入口、编辑入口和进度。

## 功能需求

- 抽出脑图上方通用工具栏，所有脑图页面复用；场景差异通过页面级配置注入。
- 小宫殿支持新建、选择已有、编辑节点、新增节点、减少节点。
- 小宫殿默认名称使用用户选中的第一个节点，不再固定使用大宫殿根下第一个节点。
- 小宫殿默认显示根节点，避免进入复习时缺少上下文。
- 宫殿和小宫殿的复习进度相互独立。
- 普通练习、小宫殿练习、小宫殿复习都复用普通宫殿正式复习的翻卡逻辑。
- 练习脑图也提供“练习 / 编辑”切换，体验对齐复习页。
- 宫殿卡片操作收敛为清晰操作区：练习、编辑、复习进度、配置、删除。

## 交互细节

- 新建小宫殿时，左键选择单个节点，右键选择该节点及其子树；再次点击已选区域则反选。
- 选完节点后显示明确的“确认新建小宫殿”按钮，并在保存成功后给出反馈。
- 小宫殿翻卡时，鼠标左右键行为和普通宫殿一致。
- 对未标记卡片不再默认自动翻开；用户将鼠标悬停在某节点上按空格，才沿该节点向下自动揭示，直到碰到被标记节点。
- 自动揭示遇到多个并列分支时应同时流通，不能只处理第一个分支。
- 点击“未完成”只保存断点并停留当前页；点击“已完成”才进入揭示剩余卡片、标红、选择推进阶段等后续流程。

## 数据与接口要求

- 小宫殿需要独立练习进度、正式复习进度、排程、日志和书架展示字段。
- 父宫殿删除时应级联处理小宫殿、排程、日志和进度。
- 父宫殿脑图结构改变后，小宫殿节点集合需要清洗；若节点为空，应进入不可复习状态而不是产生非法会话。

## 验收标准

- 小宫殿可以创建、编辑、复习、练习，并和父宫殿进度互不影响。
- 普通宫殿、小宫殿、练习、复习的翻卡行为一致。
- 所有会话页完成按钮先弹出“未完成 / 已完成”。
- 书架中父宫殿和小宫殿状态展示清楚，不再把“复习进度”和 CTA 颜色混淆。
