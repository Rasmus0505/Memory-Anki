# AI模型管理

## 背景

模型管理需求来自个人中心配置过复杂、模型调用场景不透明、生成前无法临时选择模型、前后端显示与实际调用不一致、昂贵模型误调用风险高等问题。

## 目标体验

- 用户能按模型类型管理所有模型。
- 每个 AI 场景都能看到当前实际调用的模型。
- 生成前可以选择模型和必要参数，并同步到个人中心默认配置。
- 昂贵模型调用必须透明，不能在用户不知情时发生。

## 模型分类

- 大语言模型：需标注是否具备视觉能力。
- VL / OCR 模型：用于 PDF、图片、题目识别等。
- 翻译模型：用于句子翻译、英语阅读翻译等。
- ASR 或听力相关模型：用于后续音频识别扩展。
- 自定义模型：允许新增模型名、供应商、类型和参数能力。

## 功能需求

- 个人中心按模型类型展示模型，而不是按零散场景堆叠。
- 鼠标悬浮模型类型时显示该类模型应用到哪些具体场景。
- 支持全局默认模型，也支持场景级覆盖。
- 场景覆盖应有明显颜色或标记，说明它不同于全局默认配置。
- 支持删除内置和自定义模型；删除后相关场景需要回退到有效默认模型。
- 所有 AI 生成入口都提供生成前模型和参数选择。
- 前端展示的模型名必须和后端实际调用一致。
- 支持模型参数能力，例如是否开启思考模式；仅模型支持时显示。

## 安全与配置

- 主文档不保存任何 API key。
- 新供应商接入只记录“需要配置对应供应商密钥和模型列表”。
- 密钥原文只保留在原始归档，见 `90_归档与原始素材/敏感内容索引.md`。

## 验收标准

- 能列出所有 AI 调用场景和对应模型。
- 修改个人中心配置后，实际生成场景模型名同步变化。
- 生成前临时修改模型后，下次默认记住用户选择。
- 昂贵模型场景可被用户快速识别。
