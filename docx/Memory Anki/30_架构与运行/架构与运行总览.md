# 架构与运行总览

## 核心判断

Memory Anki 未来应收敛为一个“任意学习材料 -> AI 结构化 -> 记忆宫殿 -> 题库 -> 科学复习”的统一训练引擎。英语阅读、英语听力、题库、随心模式和计时反馈都应成为这个引擎上的应用层，而不是彼此割裂的烟囱模块。

## 架构方向

- 前端：统一会话状态、脑图工具栏、反馈系统、计时器和长任务状态订阅。
- 后端：统一学习事件、AI 任务、材料来源、题库、复习调度和日志。
- 数据层：本地优先，明确版本、备份、软删除、迁移和同步冲突处理。
- AI 工作流：生成前可配置，生成中可观察，生成后可预览确认，调用结果可追溯。
- 测试：重点覆盖数据丢失、复习排程、同步、AI 任务恢复和题库写入。
