# 00 产品定位与北极星

> 上级文档：[[00_Memory Anki 未来架构愿景总纲]]

---

## 1. 产品定位

Memory Anki 应该长成一个本地优先的个人学习训练工作台。

它的核心不是“存很多知识”，也不是“生成很多题”，而是建立一个持续运转的训练闭环：

```text
知识材料
  -> 结构化
  -> 记忆宫殿
  -> 题库
  -> 复习 / 练习 / 做题
  -> 反馈
  -> 下一步推荐
```

## 2. 不应该优先变成什么

Memory Anki 不应优先变成：

- 面向公众发布的 SaaS。
- 多人协作知识库。
- 通用笔记软件。
- 单纯的 Anki 替代品。
- 单纯的 AI 题库生成器。
- 单纯的计时器或学习统计面板。

## 3. 应该优先变成什么

Memory Anki 应该是：

- 个人本地运行的学习训练工作台。
- 以“训练闭环”为核心的知识系统。
- 用记忆宫殿组织知识。
- 用题库评估掌握。
- 用复习维持记忆。
- 用 AI 降低知识生产成本。
- 让用户低负担进入心流训练。
- 长期积累个人学习数据，并反向指导下一步行动。

## 4. 产品北极星

产品北极星不是“功能数量”，而是：

> 从打开应用到进入有效训练状态的时间足够短，训练后得到的反馈足够明确，下一步足够自然。

## 5. 衡量指标

可以用这些问题判断产品是否变好：

- 打开应用后 10 秒内是否知道现在该做什么？
- 一次训练结束后是否知道哪里薄弱？
- 新材料进入系统后是否能顺畅变成训练内容？
- 是否减少了“选择页面、找任务、想流程”的认知负担？
- 是否稳定保护了个人本地数据？
- AI 是否降低了生产成本，而不是制造更多不可控内容？

## 6. 核心体验原则

### 6.1 训练优先

所有功能最终都要回答：

- 它是否帮助我更快进入学习？
- 它是否帮助我更清楚地知道哪里薄弱？
- 它是否帮助我更自然地知道下一步？

如果不能，就不是主线。

### 6.2 AI 是助教，不是黑箱主人

AI 可以主动建议，但不应该悄悄改变核心数据。

原则：

- AI 生成进入草稿。
- AI 写入需要确认。
- AI 建议需要解释原因。
- AI 操作需要可追踪。

### 6.3 本地优先，可靠性优先

个人学习数据比功能速度更重要。

原则：

- 本地数据安全优先。
- 备份优先。
- 迁移可解释。
- 不为了未来云化过早复杂化。
