## 🎯 解决方案

### 方案总览

我们需要实现以下内容：

```
缺失的页面（必须创建）：
├── MobilePalaceDetailPage.tsx        ← 宫殿详情
├── MobilePalacePracticePage.tsx      ← 宫殿练习
├── MobileFreestylePracticePage.tsx   ← 随心练习
├── MobileReviewSessionPage.tsx       ← 复习会话
├── MobileEnglishCoursePage.tsx       ← 英语课程
└── MobileEnglishReadingPage.tsx      ← 英语阅读

需要完善的功能（修改现有文件）：
├── MobileFreestylePage.tsx           ← 实现开始练习
├── MobilePalaceListPage.tsx          ← 连接真实API
├── MobileReviewPage.tsx              ← 实现开始复习
└── MobileEnglishPage.tsx             ← 实现模块跳转

需要更新的路由：
└── MobileAppRouter.tsx               ← 添加所有缺失路由
```

### 技术实现路径

#### 路径1：复用桌面端逻辑
**优点：**
- ✅ 快速实现
- ✅ 逻辑已验证
- ✅ 减少代码重复

**缺点：**
- ⚠️ 需要移动端适配
- ⚠️ UI需要重写

**推荐指数：** ⭐⭐⭐⭐⭐

#### 路径2：完全重写
**优点：**
- ✅ 完全为移动端优化

**缺点：**
- ❌ 工作量大
- ❌ 容易出bug
- ❌ 维护成本高

**推荐指数：** ⭐⭐

### 推荐方案：混合模式

```
1. 数据层 - 100%复用桌面端
   ├── API调用 - 直接使用 `@/features/*/api/`
   ├── 数据模型 - 使用 `@/features/*/model/`
   └── 业务逻辑 - 复用hooks和工具函数

2. UI层 - 移动端专属
   ├── 页面组件 - 新建 Mobile* 组件
   ├── 交互方式 - 触控优化
   └── 布局 - 全屏、卡片式

3. 路由层 - 独立配置
   └── 移动端路由树 - /m/* 独立管理
```

## 🔧 技术方案细节

### 1. 如何复用桌面端代码

#### 复用API调用
```typescript
// ✅ 好的做法：直接使用现有API
import { getPalacesApi } from '@/features/palace-catalog/api/palaceApi'

// 在移动端页面中
const loadPalaces = async () => {
  const data = await getPalacesApi()
  setPalaces(data)
}
```

#### 复用数据模型
```typescript
// ✅ 好的做法：使用现有类型
import type { Palace } from '@/features/palace-catalog/model/palace'

const [palace, setPalace] = useState<Palace | null>(null)
```

#### 复用业务逻辑
```typescript
// ✅ 好的做法：使用现有hooks
import { useFreestyleQueue } from '@/features/freestyle/hooks/useFreestyleQueue'

// 在移动端组件中直接使用
const { queue, next, current } = useFreestyleQueue()
```

## ⚠️ 常见陷阱

### 陷阱1：忘记更新路由
```typescript
// ❌ 创建了页面但忘记在路由中注册
// 结果：点击后404

// ✅ 正确做法：同时更新 MobileAppRouter.tsx
<Route path="/m/palaces/:id" element={<MobilePalaceDetailPage />} />
```

### 陷阱2：导入路径错误
```typescript
// ❌ 错误：相对路径
import { getPalacesApi } from '../../../palace-catalog/api'

// ✅ 正确：使用 @ 别名
import { getPalacesApi } from '@/features/palace-catalog/api/palaceApi'
```

### 陷阱3：类型不匹配
```typescript
// ❌ 错误：使用自己定义的类型
interface MyPalace { id: number, name: string }

// ✅ 正确：使用现有类型
import type { Palace } from '@/features/palace-catalog/model/palace'
```
