# 复习态灰卡/绿卡/绿线补齐方案

## Summary
补上当前复习/练习页缺失的状态反馈，让导图在保留宿主界面和功能键的前提下，具备这套复习语义：

- 未翻开的卡片是灰卡
- 翻开内容后的卡片变绿
- 当某卡片的所有一级子卡片都已翻开时，该卡片到这些一级子卡片的直接连线变绿

本次方向已锁定为“保宿主优先”：
- 不放弃现在这套宿主导图界面
- 在宿主节点数据能力内补样式状态
- 优先稳定可用，不再回到会把工具栏弄没的自绘整图方案

## Key Changes
### 1. 给复习态节点补显式视觉状态映射
- `MindMapReviewFlow` 继续维护 `hidden / placeholder / revealed` 三态，但不再只改文字。
- 在构造复习态 `editor_doc` 时，把节点状态同步写入宿主可识别的数据字段：
  - `placeholder` 节点写入灰卡样式
  - `revealed` 节点写入绿卡样式
  - 根节点保留已翻开态，但视觉层级比普通绿卡更突出
- 当前的“待回忆”占位文字保留作为 v1 占位表现，不再强求纯空白卡，以避免宿主把节点压缩得不可点。

### 2. 用宿主节点数据驱动卡片颜色
- 利用宿主已有的节点数据扩展字段能力，在复习态节点上写入卡片样式相关数据，而不是只改 `text`。
- 灰卡样式固定为：
  - 低饱和灰底
  - 深一点的灰边框
  - 内容不显示备注
  - 占位文字保留简短提示
- 绿卡样式固定为：
  - 柔和绿色底
  - 稍深绿色边框
  - 显示真实标题与备注
- 复习态切换节点时，这些样式只在复习/练习传给宿主的临时 `editor_doc` 生效，不回写正式宫殿编辑数据。

### 3. 按一级子卡完成态驱动连线变绿
- 在前端基于 `revealMap` 计算“某父节点的所有一级子节点都为 `revealed`”这一完成条件。
- 当条件成立时，把该父节点到这些直接子节点的边样式标记为绿色完成态。
- 判定只看直接 children：
  - 不要求二级及更深节点完成
  - 子节点是否还有后代，不影响这一层变绿
- 只改父子直连线，不扩散到整棵子树，避免反馈过度。

### 4. 保持宿主稳定性，不引入新的重置问题
- 灰卡/绿卡/绿线样式变更继续走“稳定 iframe + 同步 editorState”路径，不能重新引入 iframe 重建。
- 每次 reveal 只更新宿主数据，不重置：
  - 当前缩放和平移视图
  - 当前全屏状态
  - 当前导图宿主实例
- 练习页和正式复习页共用同一套复习态节点/边状态映射，避免两套视觉规则漂移。

### 5. 交互语义保持不变
- 单击 `placeholder`：翻开内容，卡片变绿
- 单击 `revealed`：放出一个一级子卡，占位灰卡出现
- 双击 `revealed`：放出一个下一个真实同级卡，占位灰卡出现
- 这次只补视觉反馈，不改当前 reveal 顺序、评分逻辑、正式复习提交语义

## Public Interfaces
- 不新增后端接口，不修改 `/review/session/:id`、`/palaces/:id/practice` 路由语义。
- 前端内部仍沿用现有 `RevealState = hidden | placeholder | revealed`。
- `MindMapReviewFlow` 的输入输出语义不变，只增强“构造复习态 editor_doc”时附加的节点/边样式字段。
- `MindMapFrame` 与宿主桥接不需要新增后端依赖，只需要继续支持复习态数据同步。

## Test Plan
1. 初始进入练习/正式复习页：
   只看到根节点，根节点为已翻开态，其他节点不出现。
2. 单击根节点后：
   只出现一个一级子卡，占位卡应明确呈现灰卡状态。
3. 单击灰卡后：
   该卡片翻开为真实内容，并切换为绿色卡片。
4. 单击已翻开的绿卡后：
   只放出一个一级子灰卡，不提前出现孙卡。
5. 双击已翻开的绿卡后：
   只放出同一父节点下的下一个真实同级灰卡，不跳序、不重复。
6. 连线完成态：
   当某父节点的所有一级子卡都已翻开后，该父节点到这些直接子卡的连线全部变绿。
7. 二级未完成场景：
   一级子卡都已翻开但孙卡未翻开时，父到子这一层仍应保持绿色。
8. 稳定性验证：
   连续翻牌过程中不闪屏、不重建 iframe、不退出全屏、不跳回根节点视图。
9. 练习与正式复习一致性：
   两个页面的灰卡/绿卡/绿线规则一致，只在评分提交后行为不同。

- 根节点默认视为已翻开节点，但保留比普通绿卡更高的视觉权重。
