### plan需求输入

## 需求7 — 宫殿脑图核心链路重构与调试增强

### 原始输入
对项目结构做一轮全方位优化，重点是解耦、高度模块化、精简复用、方便后期删减扩展、增强调试信息并清理无用代码；尤其要优先解决“转脑图”和“翻卡模式”的重复实现问题。

### plan需求输入
围绕宫殿与脑图核心域做结构重整：统一脑图宿主桥接、宫殿编辑页编排、导入链路，以及宫殿内练习和正式复习的翻卡规则，尽量保持现有路由和主要接口行为稳定；同时补强前后端可复制的诊断信息，让用户在失败时能直接把 request_id、job_id、阶段、错误码和关键上下文发给 AI 排障，并把宫殿域相关的超大文件、重复工具和已确认冗余代码收口清理。
## 需求8 — Monorepo 深度重构

### 原始输入
用户要求直接实施一套 Monorepo 深度重构方案，目标是把当前项目从名义模块化、实际旧单体的状态，重构为真实模块化后端、独立的 web 与 admin 双前端应用、清晰的静态资源构建与运行时边界，并清理兼容层、死代码、构建产物和跨模块乱依赖。

### plan需求输入
对当前 monorepo 执行破坏式但可维护的深度重构：后端统一收敛到明确的 bounded contexts 和分层结构，前端将 admin 从 web 中彻底拆出为独立应用，公共 contract 与共享能力集中到 packages，所有业务 API 收口到 /api/v1/<context>，同时删除历史兼容壳、无效目录、回写源码树的静态同步链路，以及被跟踪的构建产物和临时验证目录。
